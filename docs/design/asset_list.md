## 1. 아트 스타일 가이드

### 1-1. 비주얼 방향성

던전앤파이터(DNF)의 클래식 도트 아트 스타일을 패러디하되, 과장된 비율과 코믹한 표현으로 차별화한다. 원작의 "호쾌한 타격감"을 시각적으로 재현하면서도, 방치형 게임에 맞는 가독성 높은 디자인을 추구한다.

### 1-2. 픽셀 아트 사양

| 항목 | 사양 | 비고 |
|------|------|------|
| 기본 단위 | **32x32px** | 캐릭터/몬스터 1타일 기준 |
| 캐릭터 크기 | **64x64px** (2타일) | 무기 포함 시 96x64px까지 허용 |
| 몬스터 크기 | 32x32 ~ 128x128px | 종류에 따라 가변 |
| 보스 구슬 크기 | **128x128px** | 파괴 이펙트 포함 시 192x192px |
| PPU (Pixels Per Unit) | **32** | Unity 스프라이트 기본 설정 |
| 필터 모드 | **Point (no filter)** | 픽셀 아트 선명도 유지 |
| 압축 | **None** 또는 **Crunch (Low)** | 아틀라스 적용 시 Crunch 허용 |


### 1-4. DNF 패러디 미학 규칙

- **과장된 실루엣**: 무기는 캐릭터 몸보다 크게, 이펙트는 화면을 덮을 정도로
- **투명 외곽선**: 스프라이트 외곽에 1px 진한 남색 외곽선 (가독성 확보)
- **피격 강조**: 몬스터 피격 시 흰색 플래시(2프레임) + 넉백 모션
- **그림자**: 캐릭터/몬스터 발밑에 원형 그림자 (별도 스프라이트, 반투명)
- **코믹 요소**: 개 얼굴 구슬은 귀여우면서도 위협적인 표정, 눈동자가 움직이는 연출

## 4. 개 얼굴 구슬

### 4-1. 디자인 콘셉트

게임의 아이콘적 존재. 시바견(도지) 느낌의 얼굴이 투명한 황금빛 구슬 안에 부유하는 형태. 표정이 살아있으며, 피격 시 점점 화난 표정으로 변한다. DNF의 "골렘의 핵" 같은 역할이지만, 비주얼은 귀엽고 코믹하게.

## 12. 에셋 네이밍 컨벤션

### 12-1. 기본 규칙

- 모든 파일명은 **영문 소문자 + 밑줄(snake_case)**
- 공백, 한글, 특수문자 사용 금지
- 접두사로 에셋 유형을 구분

### 12-2. 접두사 규칙

| 접두사 | 에셋 유형 | 예시 |
|--------|----------|------|
| `spr_` | 스프라이트 (캐릭터/몬스터/오브젝트) | `spr_player_run_01.png` |
| `bg_` | 배경 이미지 | `bg_dark_far_01.png` |
| `tile_` | 타일 스프라이트 | `tile_dark_floor_01.png` |
| `deco_` | 배경 장식 | `deco_dark_torch.png` |
| `fx_` | 이펙트/파티클 | `fx_hit_normal_01.png` |
| `ui_` | UI 에셋 | `ui_btn_primary_normal.png` |
| `icon_` | 아이콘 | `icon_gold.png` |
| `font_` | 폰트/폰트 스프라이트 | `font_dmg_normal.png` |
| `bgm_` | 배경음악 | `bgm_dungeon_01.ogg` |
| `sfx_` | 효과음 | `sfx_hit_normal.wav` |
| `anim_` | 애니메이션 컨트롤러/클립 | `anim_player_run.anim` |
| `mat_` | 머터리얼 | `mat_additive_particle.mat` |
| `atlas_` | 스프라이트 아틀라스 | `atlas_player.spriteatlas` |
| `so_` | ScriptableObject | `so_monster_goblin.asset` |
| `prefab_` | 프리팹 | `prefab_monster_goblin.prefab` |

### 12-3. 스프라이트 시트 네이밍

```
spr_{주체}_{상태}_{프레임번호}.png

예시:
  spr_player_idle_00.png ~ spr_player_idle_03.png
  spr_player_run_00.png ~ spr_player_run_05.png
  spr_player_attack_combo1_00.png ~ spr_player_attack_combo1_03.png
  spr_goblin_death_00.png ~ spr_goblin_death_03.png
  spr_orb_idle_00.png ~ spr_orb_idle_05.png
```

또는 스프라이트 시트(단일 이미지)로 제작 시:

```
sheet_{주체}_{상태}.png

예시:
  sheet_player_run.png        (6프레임 가로 배열, 384x64px)
  sheet_goblin_idle.png       (4프레임 가로 배열, 128x32px)
  sheet_orb_destruction.png   (10프레임 가로 배열, 1920x192px)
```

### 12-4. UI 에셋 네이밍

```
ui_{유형}_{이름}_{상태}.png

예시:
  ui_btn_primary_normal.png
  ui_btn_primary_pressed.png
  ui_btn_primary_disabled.png
  ui_panel_popup.png
  ui_bar_hp_frame.png
  ui_bar_hp_fill_player.png
```

### 12-5. 이펙트 네이밍

```
fx_{분류}_{이름}_{프레임번호}.png

예시:
  fx_hit_normal_00.png ~ fx_hit_normal_03.png
  fx_skill_slash_wave_00.png ~ fx_skill_slash_wave_05.png
  fx_orb_shatter_00.png ~ fx_orb_shatter_07.png
```

### 12-6. 사운드 네이밍

```
bgm_{장면/용도}.ogg
sfx_{분류}_{이름}.wav

예시:
  bgm_dungeon_01.ogg
  bgm_boss.ogg
  sfx_hit_normal.wav
  sfx_skill_slash.wav
  sfx_ui_click.wav
  sfx_orb_shatter.wav
```
