## 1. 재화 종류 정의

### 1-1. 골드 (Gold)

| 항목 | 내용 |
|------|------|
| **설명** | 기본 재화. 몬스터 처치 시 획득하는 일반 통화 |
| **아이콘** | 금화 (동전 모양) |
| **저장 한도** | 없음 (무제한 축적 가능, BigInteger 표기 사용) |

**획득 경로:**

| 경로 | 비율 | 비고 |
|------|------|------|
| 몬스터 처치 | 주요 수입원 (약 70%) | 스테이지별 스케일링 |
| 스테이지 클리어 보너스 | 약 15% | 구슬 파괴 시 일괄 지급 |
| 오프라인 보상 | 약 10% | 경과시간 * 효율계수 |
| 업적/미션 보상 | 약 5% | 일회성 또는 일일 미션 |

**소비처:**

| 소비처 | 비중 | 설명 |
|--------|------|------|
| 스탯 강화 | 약 50% | ATK, ATK_SPEED, CRIT_RATE, CRIT_DMG, MOVE_SPEED 레벨업 |
| 스킬 레벨업 | 약 30% | 스킬 데미지/쿨타임 개선 |
| 장비 강화 | 약 15% | 장비 레벨업, 재련 |
| 기타 | 약 5% | 소모품 구매, 즉시 부활 등 |

**전생(Prestige) 시 동작:** 전생 시 골드 **전액 초기화**

## 2. 골드 수입 테이블

### 2-1. 기본 공식

모든 골드 획득량은 다음 스케일링 공식을 따른다:

```
GoldPerKill(stage) = BaseGold * (1 + stage * 0.12) ^ 1.15

여기서:
  BaseGold = 11 (몬스터 5종 평균, 개별 값은 monster_data.md 참조)
  stage = 현재 스테이지 번호
```

이 공식은 선형보다 빠르지만 지수보다 느린 **준지수(quasi-exponential)** 증가를 만든다.

### 2-2. 스테이지별 몬스터 처치 골드

| 스테이지 | GoldPerKill 계산 | 몬스터당 골드 | 방당 몬스터 수 | 방당 골드 |
|----------|-----------------|--------------|---------------|----------|
| 1 | `11 * (1 + 1*0.12)^1.15` | **13** | 3마리 | **39** |
| 10 | `11 * (1 + 10*0.12)^1.15` | **27** | 3마리 | **81** |
| 50 | `11 * (1 + 50*0.12)^1.15` | **103** | 6마리 | **618** |
| 100 | `11 * (1 + 100*0.12)^1.15` | **210** | 8마리 | **1,680** |
| 500 | `11 * (1 + 500*0.12)^1.15` | **1,244** | 8마리 | **9,952** |

> **참고:** 방당 몬스터 수는 스테이지가 높아질수록 점진적으로 증가 (최소 3, 최대 8)
> `MonstersPerRoom(stage) = min(8, 3 + floor(stage / 15))`

### 2-3. 스테이지 클리어 골드 (구슬 파괴 보너스)

```
StageClearGold(stage) = GoldPerKill(stage) * RoomsPerStage * MonstersPerRoom * 0.25

  즉, 해당 스테이지 총 몬스터 골드의 25%를 보너스로 지급
```

| 스테이지 | 방 수 | 총 몬스터 골드 | 클리어 보너스 (25%) | 스테이지 총 골드 |
|----------|------|---------------|-------------------|----------------|
| 1 | 5 | 165 | 41 | **206** |
| 10 | 5 | 375 | 94 | **469** |
| 50 | 7 | 3,906 | 977 | **4,883** |
| 100 | 8 | 11,328 | 2,832 | **14,160** |
| 500 | 8 | 56,192 | 14,048 | **70,240** |

> **방 수 공식:** `RoomsPerStage(stage) = min(8, 5 + floor(stage / 20))`

### 2-4. 분당 골드 획득량

클리어 속도 가정:
- 1방 클리어 시간: 초반 약 8초, 후반 약 15초 (몬스터 HP 증가로 인한 둔화)
- 스테이지 전환 시간: 2초

```
TimePerStage(stage) = RoomsPerStage * TimePerRoom(stage) + TransitionTime
TimePerRoom(stage)  = BaseRoomTime * (1 + stage * 0.005)  (초 단위, 최대 20초)
  여기서 BaseRoomTime = 8초
```

| 스테이지 | 방 클리어 시간 | 스테이지 시간 | 스테이지 총 골드 | **분당 골드** |
|----------|--------------|-------------|----------------|-------------|
| 1 | 8.04초 | 42초 | 206 | **294** |
| 10 | 8.4초 | 44초 | 469 | **640** |
| 50 | 10초 | 72초 | 4,883 | **4,069** |
| 100 | 12초 | 98초 | 14,160 | **8,671** |
| 500 | 20초 (캡) | 162초 | 70,240 | **26,015** |

## 3. 골드 지출 테이블

### 3-1. 스탯 강화 비용

스탯 강화 비용 상세는 [character_growth.md](character_growth.md) 참조. 본 문서의 경제 분석에서는 ATK 기준 평균 비용을 사용함.

### 3-2. 스킬 레벨업 비용

스킬 레벨업은 스탯보다 비용이 높지만, 전투력 상승 효과가 크다.

> **참고:** 상세 스킬별 비용, 효과, 최대 레벨 등은 [skill_data.md](skill_data.md) 참조.
> 아래 수치는 경제 밸런스 분석을 위한 간략 예시이며, 실제 적용 값은 skill_data.md가 기준.

```
SkillUpgradeCost(level) = SkillBaseCost * (1.12 ^ level)

  여기서:
    SkillBaseCost = 스킬 등급별 기본 비용 (상세 비용은 skill_data.md 참조)
```

**스킬 등급별 비용 (간략 예시):**

| 스킬 등급 | SkillBaseCost (예시) | 레벨당 효과 | 최대 레벨 |
|----------|---------------------|-----------|----------|
| 기본 평타 콤보 | 100 | +8% 데미지 | 50 |
| 범위 스킬 | 200 | +10% 데미지, -0.5초 쿨타임 | 30 |
| 궁극기 | 500 | +15% 데미지, -1초 쿨타임 | 20 |

**스킬 레벨별 비용 (범위 스킬 기준, BaseCost=200 간략 예시):**

| 레벨 | 단일 업그레이드 비용 | 누적 비용 |
|------|-------------------|----------|
| 1 | 224 | 224 |
| 5 | 352 | 1,431 |
| 10 | 621 | 4,212 |
| 15 | 1,094 | 10,561 |
| 20 | 1,929 | 24,654 |
| 25 | 3,400 | 55,038 |
| 30 | 5,993 | 119,854 |

### 3-3. 파워 고원(Power Plateau)별 총 필요 골드

**파워 고원** = 다음 스테이지 구간을 돌파하기 위해 필요한 최소 전투력 수준

| 파워 고원 | 목표 스테이지 구간 | 필요 스탯 레벨 (평균) | 필요 스킬 레벨 (평균) | **총 필요 골드** |
|----------|------------------|---------------------|---------------------|----------------|
| 초반 1 | 1~10 | ATK 5, 나머지 3 | 평타 3, 범위 1 | **~2,000** |
| 초반 2 | 10~25 | ATK 10, 나머지 7 | 평타 7, 범위 3 | **~8,500** |
| 중반 1 | 25~50 | ATK 20, 나머지 15 | 평타 12, 범위 7, 궁극 3 | **~45,000** |
| 중반 2 | 50~100 | ATK 35, 나머지 25 | 평타 20, 범위 12, 궁극 7 | **~180,000** |
| 후반 1 | 100~200 | ATK 50, 나머지 40 | 평타 30, 범위 18, 궁극 12 | **~750,000** |
| 후반 2 | 200~500 | ATK 75, 나머지 55 | 평타 40, 범위 25, 궁극 16 | **~3,500,000** |
| 극후반 | 500+ | ATK 100+, 나머지 75+ | 전부 최대 | **~15,000,000+** |

## 4. 수입/지출 밸런스 분석

### 4-1. 설계 목표

| 게임 단계 | 스테이지 구간 | 업그레이드 간 목표 시간 | 체감 |
|----------|-------------|---------------------|------|
| 초반 (Early) | 1~25 | **2~5분** | 빠른 성장, 즉각 피드백 |
| 중반 (Mid) | 25~100 | **10~20분** | 의사결정 중요, 적절한 긴장감 |
| 후반 (Late) | 100~500 | **30~60분** | 오프라인 보상 의존도 증가 |
| 극후반 (End) | 500+ | **2~8시간** | 전생 유도, 광고/보석 소비 |

### 4-2. 구간별 밸런스 검증

#### 초반 (스테이지 1~10)

```
분당 골드 수입: ~294 ~ 640 (평균 ~467)
다음 업그레이드 비용: ~500~2,000
업그레이드 소요 시간: 500/467 ~ 2,000/640 = 1.1분 ~ 3.1분
```

**판정:** 목표 범위 내 (2~5분). 초반은 빠르게 성장하는 쾌감을 제공.
여러 스탯을 동시에 올려야 하므로 실제 체감은 3~5분 간격.

#### 초반~중반 전환 (스테이지 10~25)

```
분당 골드 수입: ~640 ~ 1,500 (평균 ~1,070)
초반2 고원 필요 골드: ~8,500
이전 고원에서 소비 후 잔액: ~0 (전부 투자했다 가정)
순수 파밍 시간: 8,500 / 1,070 ≈ 7.9분
```

**판정:** 목표 범위 상단 (2~5분 초과, 10분 미만). 중반 진입 전 적절한 벽.
장비 드롭과 오프라인 보상을 포함하면 체감은 5~6분 수준.

#### 중반 (스테이지 25~50)

```
분당 골드 수입: ~1,500 ~ 4,069 (평균 ~2,785)
중반1 고원 필요 골드: ~45,000
이전 고원 투자 후 추가 필요: ~45,000 - 8,500 = 36,500
순수 파밍 시간: 36,500 / 2,785 ≈ 13.1분
```

**판정:** 목표 범위 내 (10~20분). 중반 진입에 적합한 난이도.

#### 중반~후반 전환 (스테이지 50~100)

```
분당 골드 수입: ~4,069 ~ 8,671 (평균 ~6,370)
중반2 고원 필요 골드: ~180,000
추가 필요: ~180,000 - 45,000 = 135,000
순수 파밍 시간: 135,000 / 6,370 ≈ 21.2분
```

**판정:** 목표 범위 약간 초과 (10~20분). 후반 진입 직전의 의미 있는 벽.
오프라인 보상과 장비 효과를 포함하면 체감은 15~18분 수준.

#### 후반 (스테이지 100~200)

```
분당 골드 수입: ~8,671 ~ 16,000 (평균 ~12,335)
후반1 고원 필요 골드: ~750,000
추가 필요: ~750,000 - 180,000 = 570,000
순수 파밍 시간: 570,000 / 12,335 ≈ 46.2분
```

**판정:** 목표 범위 내 (30~60분). 오프라인 보상이 핵심 보조 수입원.

#### 극후반 (스테이지 200~500)

```
분당 골드 수입: ~16,000 ~ 26,015 (평균 ~21,008)
후반2 고원 필요 골드: ~3,500,000
추가 필요: ~3,500,000 - 750,000 = 2,750,000
순수 파밍 시간: 2,750,000 / 21,008 ≈ 131분 (약 2.2시간)
```

**판정:** 목표 범위 내 (2~8시간). 전생 시스템 유도 구간.
오프라인 보상 + 광고 부스트 + 전생 영구 버프로 체감 단축.

## 5. 구슬 조각 경제

### 5-1. 구슬 조각 획득 공식

```
OrbFragments(stage) = BaseFragments * (1 + floor(stage / 10) * 0.25)

  여기서:
    BaseFragments = 5 (스테이지 1~9 기준)
```

| 스테이지 구간 | 보스당 구슬 조각 | 정예 몬스터 드랍 (8~15% 확률, 몬스터별 상이) | 비고 |
|-------------|-----------------|--------------------------|------|
| 1~9 | 5 | 1 | 초반 축적 |
| 10~19 | 6 | 1 | 첫 가챠 목표 |
| 20~49 | 8 ~ 10 | 1~2 | 정기적 가챠 |
| 50~99 | 11 ~ 16 | 2 | 특수 강화 접근 |
| 100~199 | 18 ~ 29 | 2~3 | 가챠 + 특수 강화 병행 |
| 200~499 | 30 ~ 66 | 3 | 고급 가챠 접근 |
| 500+ | 68+ | 3~5 | 최상위 장비 도전 |

### 5-2. 장비 가챠 비용 및 확률

**일반 가챠 (구슬 조각 사용):**

| 항목 | 비용 | 비고 |
|------|------|------|

### 5-3. 가챠 기대값 분석

**전설 장비 1개 획득까지 기대 소요 (천장 시스템 포함):**

```
대천장(Hard Pity) = 100회 (Legendary 확정)
필요 구슬 조각 = 100 * 50 = 5,000개 (단일 뽑기 기준)
                 또는 450 * 10 = 4,500개 (10연차 기준)

스테이지 50 기준 보스당 11조각:
  5,000 / 11 = 455 스테이지 클리어 필요
  스테이지 50에서의 클리어 시간: 약 72초/스테이지
  455 * 72 = 32,760초 ≈ 546분 ≈ 9.1시간

스테이지 100 기준 보스당 18조각:
  5,000 / 18 = 278 스테이지 클리어 필요
  스테이지 100에서의 클리어 시간: 약 98초/스테이지
  278 * 98 = 27,244초 ≈ 454분 ≈ 7.6시간

참고: 1% 확률 기대값(100회)보다 일찍 뜰 수 있으며,
소천장(50회 이후 Epic 확률 2배)이 중간 보상감을 제공.
```

**의도:** 전설 장비는 "특별한 이벤트"로 느껴져야 하며, 무과금 기준 7~9시간에
천장 도달 가능. 오프라인 보상(구슬 조각 30% 효율)과 정예 몬스터 드랍을
포함하면 실제 체감은 5~7시간 수준. 장기 목표로서의 동기 부여 역할.

## 7. 인플레이션 관리

### 7-1. 인플레이션 문제 정의

무한 스테이지 시스템에서 골드 수입이 무한히 증가하면 숫자가 의미를 잃고 게임 경제가 붕괴된다.

### 7-2. 인플레이션 억제 메커니즘

#### (A) 지수 비용 증가

골드 수입은 준지수적(`^1.15`)이지만, 스탯 강화 비용은 순수 지수(`1.08^level`)로 증가.

```
수입 성장률: O(stage^1.15)    -- 준지수 (느림)
비용 성장률: O(1.08^level)    -- 순수 지수 (빠름)

level이 stage에 비례하여 증가하므로, 고스테이지에서 비용 증가가
수입 증가를 추월 → 자연 인플레이션 억제
```

#### (B) 골드 싱크(소멸 장치) 목록

| 싱크 종류 | 설명 | 효과 |
|----------|------|------|
| 전생 리셋 | 전생 시 골드 전액 초기화 | 가장 강력한 싱크 |
| 실패 비용 | 장비 강화 실패 시 재료 소모 | 지속적 골드/조각 소비 |
| 스탯 리스펙 비용 | 스탯 재분배 시 20% 세금 | 잘못된 투자 교정 비용 |
| 소모품 | 일시 버프 아이템 구매 | 지속적 소비 유도 |
| 장비 수리 (선택) | 장비 내구도 소모 시 수리비 | 반복적 소비 |

#### (C) 전생 시스템의 인플레이션 제어 역할

```
전생 전 상태:
  - 스테이지 200, 분당 골드 20,000
  - 스탯 레벨 평균 55, 업그레이드 비용 ~500,000
  - 업그레이드 간격: 25분 (체감 둔화)

전생 후 상태:
  - 스테이지 1 리셋, 골드 0
  - 영구 버프: ATK +20%, Gold +15% (누적)
  - 분당 골드: 294 * 1.15 = 338 (초반이지만 버프 적용)
  - 빠른 초반 재성장 → 쾌감 회복

핵심 효과: 전생마다 경제 "리셋" → 인플레이션 주기적 해소
```

#### (D) 소프트 캡(Soft Cap) 시스템

| 적용 대상 | 캡 방식 | 수치 |
|----------|--------|------|
| 분당 골드 | 하드캡 없음, 전생 유도 | 전생 시 리셋 |
| CRIT_RATE | 하드캡 80% | 초과 투자 방지 |
| ATK_SPEED | 소프트캡 (200% 이후 수확 체감) | 200% 이후 레벨당 +1% |
| MOVE_SPEED | 소프트캡 (300% 이후 체감) | 게임 플레이 가능 범위 유지 |

### 7-3. 숫자 표기 스케일링

인플레이션이 시각적으로 문제되지 않도록 약식 표기 적용:

```
1,000        → 1.00K
1,000,000    → 1.00M
1,000,000,000 → 1.00B
10^12        → 1.00T
10^15        → 1.00Qa
10^18        → 1.00Qi
```

## 8. 오프라인 보상 경제 영향

### 8-1. 오프라인 보상 기본 공식

```csharp
// 기본 공식
float offlineGold = goldPerSecond * offlineDuration * efficiencyCoefficient;

// 변수 정의
goldPerSecond = 현재 스테이지의 분당 골드 / 60
offlineDuration = 마지막 접속 이후 경과 시간 (초)
efficiencyCoefficient = 0.50 (기본값, 50%)
```

### 8-2. 효율 계수 (Efficiency Coefficient) 설계

| 조건 | 효율 계수 | 비고 |
|------|----------|------|
| 기본 | 0.50 (50%) | 온라인 대비 절반 |
| 광고 시청 시 | 1.00 (100%) | 30초 광고 → 2배 보상 |
| 보석 사용 시 | 1.00 (100%) | 30보석 → 즉시 수령 |
| 전생 영구 버프 | +0.05 per 전생 횟수 | 최대 +0.25 (5회분) |
| VIP/구독 (미래) | 0.80 (80%) | 유료 구독 보너스 |

**최대 효율 계수:** 0.50 + 0.25 = 0.75 (기본 + 전생 버프, 광고 미포함)

### 8-3. 오프라인 시간 캡

| 항목 | 수치 | 이유 |
|------|------|------|
| 최소 오프라인 시간 | 5분 | 너무 짧은 이탈은 보상 불필요 |
| 최대 누적 시간 | 12시간 (43,200초) | 수면+출퇴근 시간 기준 (idle_system.md 참조) |
| 광고 2배 적용 최대 | 4시간 | 광고 보상 과다 방지 |

### 8-4. 오프라인 보상 구체 수치

**스테이지 50 기준 (분당 골드: 4,069):**

```
goldPerSecond = 4,069 / 60 = 67.8

오프라인 1시간 (3,600초):
  기본: 67.8 * 3,600 * 0.50 = 122,040 골드
  광고: 67.8 * 3,600 * 1.00 = 244,080 골드

오프라인 4시간:
  기본: 67.8 * 14,400 * 0.50 = 488,160 골드
  광고: 67.8 * 14,400 * 1.00 = 976,320 골드

오프라인 12시간 (캡, 43,200초):
  기본: 67.8 * 43,200 * 0.50 = 1,464,480 골드
  광고: 67.8 * 14,400 * 1.00 + 67.8 * 28,800 * 0.50
       = 976,320 + 976,320 = 1,952,640 골드
  (광고 2배는 4시간까지만 적용)
```

### 8-5. 오프라인 보상의 밸런스 영향

**핵심 원칙:** 오프라인 보상은 온라인 플레이를 **보조**하되 **대체**하면 안 된다.

```
온라인 1시간 파밍 (스테이지 50): 4,069 * 60 = 244,140 골드
오프라인 1시간 보상 (기본):       122,040 골드 (온라인의 50%)
오프라인 1시간 보상 (광고):       244,080 골드 (온라인과 동일)
```

**의도:**
- 기본 오프라인은 온라인의 50% → 온라인 플레이 동기 유지
- 광고 시청 시 온라인 수준 → 광고 시청 동기 부여
- 12시간 캡 → 과도한 방치 보상 방지 (24시간 방치해도 12시간 분량만)

### 8-6. 구슬 조각 오프라인 보상

구슬 조각은 보스 처치 기반이므로 별도 계산:

```
오프라인 구슬 조각 = floor(offlineDuration / avgStageClearTime) * OrbFragments(stage) * 0.30

  여기서:
    avgStageClearTime = 해당 스테이지 클리어 시간 (초)
    0.30 = 오프라인 구슬 효율 (온라인의 30%)
```

**의도:** 구슬 조각은 희귀 재화이므로 오프라인 효율을 골드보다 낮게 설정 (30%).

## 9. 광고 수익화 포인트

### 9-1. 광고 배치 전략 원칙

1. **자발적 선택:** 모든 광고는 강제가 아닌 "선택형"
2. **명확한 보상:** 광고 시청 전 보상 내용을 명확히 표시
3. **적정 빈도:** 과도한 광고는 이탈을 유발
4. **가치 제공:** 광고 시청이 "의미 있는 보상"이어야 시청률 유지

### 9-2. 광고 배치 포인트

| 보석 획득 | 메인 화면 버튼 | 보석 5개 | 10회 | 30초 |
| 골드 부스트 | 강화 탭 근처 | 30분 골드 2배 | 2회 | 30초 |

### 9-3. 광고 보상 수치 상세

#### (A) 오프라인 보상 2배

```
일일 최대 효과 (스테이지 50 기준):
  기본 오프라인 12시간: 1,464,480 골드
  광고 3회 시청 (4시간분 2배): +976,320 골드
  증가율: +67% (총 오프라인 보상 기준)
```

#### (B) 보석 광고

```
일일 최대: 5보석 * 10회 = 50보석
광고 총 시간: 10 * 30초 = 5분
보석 가치 환산:
  50보석 ≈ 골드 부스트 2.5회 (30분 * 2.5 = 75분 2배)
  또는 프리미엄 가챠 1회 (50보석)
```

#### (C) 골드 부스트

```
일일 최대: 2회 * 30분 = 60분 골드 2배
스테이지 50 기준 추가 골드: 4,069 * 60 = 244,140 골드
광고 총 시간: 2 * 30초 = 1분
```

#### (D) 가챠 추가 뽑기

```
일일 최대: 3회 추가 뽑기
구슬 조각 가치 환산: 3 * 50 = 150 구슬 조각 절약
```

| 신호 | 원인 추정 | 조정 방향 |
