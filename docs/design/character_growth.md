## 1. 기본 스탯 정의

### 1-1. 스탯 총괄표

| 스탯 | 필드명 | 설명 | 기본값 | 최솟값 | 최댓값 (하드캡) | 단위 |
|------|--------|------|--------|--------|-----------------|------|
| 공격력 | ATK | 1회 타격당 기본 피해량 | 10 | 10 | 999,999 | 정수 |
| 공격 속도 | ATK_SPEED | 초당 공격 횟수 | 1.0 | 0.5 | 10.0 | 회/초 |
| 치명타 확률 | CRIT_RATE | 공격 시 치명타 발동 확률 | 5% | 0% | 80% | 퍼센트 |
| 치명타 배율 | CRIT_DMG | 치명타 시 피해 배율 | 150% | 100% | 500% | 퍼센트 |
| 이동 속도 | MOVE_SPEED | 캐릭터 수평 이동 속도 | 3.0 | 1.0 | 15.0 | 유닛/초 |
| 체력 | HP | 캐릭터 최대 체력 (선택적) | 100 | 100 | 999,999 | 정수 |

### 1-2. 스탯별 상세 설명

#### ATK (공격력)

- **역할**: 모든 데미지 계산의 기반이 되는 핵심 스탯
- **효과 공식**: `FinalDamage = ATK * SkillMultiplier * (IsCrit ? CRIT_DMG : 1.0)`
- **설계 의도**: 선형적으로 증가하여 플레이어가 성장을 직관적으로 느끼도록 함
- **레벨당 증가량**: `ATK(Lv) = BaseATK + (Lv - 1) * ATK_PER_LEVEL`
  - `ATK_PER_LEVEL = 3` (레벨 1~50), `5` (레벨 51~100)

#### ATK_SPEED (공격 속도)

- **역할**: 단위 시간당 공격 횟수를 결정. DPS에 직접 곱연산
- **효과 공식**: `AttackInterval = 1.0 / ATK_SPEED` (초)
- **설계 의도**: 초반에는 느리게 시작해서 후반에 빠른 타격감을 선사
- **레벨당 증가량**: `ATK_SPEED(Lv) = BaseSpeed + (Lv - 1) * 0.05`
- **소프트캡 적용**: 5.0 이상부터 증가량 절반 (0.025/레벨)

#### CRIT_RATE (치명타 확률)

- **역할**: 평균 DPS를 확률적으로 증가시키는 보조 스탯
- **효과 공식**: `EffectiveDPSMultiplier = 1 + CRIT_RATE * (CRIT_DMG - 1)`
- **설계 의도**: 중반 이후 DPS 스케일링의 핵심 수단
- **레벨당 증가량**: `CRIT_RATE(Lv) = BaseCritRate + (Lv - 1) * 0.8%`
- **하드캡**: 80% (100%에 도달하면 치명타의 의미가 사라지므로)

#### CRIT_DMG (치명타 배율)

- **역할**: 치명타 발동 시 추가 피해 배율
- **효과 공식**: `CritDamage = ATK * CRIT_DMG`
- **설계 의도**: CRIT_RATE와 시너지하여 후반 스케일링 곡선을 만듦
- **레벨당 증가량**: `CRIT_DMG(Lv) = BaseCritDmg + (Lv - 1) * 2.5%`
- **하드캡**: 500%

#### MOVE_SPEED (이동 속도)

- **역할**: 방 사이 이동 시간 단축, 실질적 스테이지 클리어 속도에 영향
- **효과 공식**: `RoomTraversalTime = RoomWidth / MOVE_SPEED` (초)
- **설계 의도**: DPS에 직접 기여하지 않으나 체감 효율에 큰 영향
- **레벨당 증가량**: `MOVE_SPEED(Lv) = BaseSpeed + (Lv - 1) * 0.1`
- **소프트캡 적용**: 8.0 이상부터 증가량 절반 (0.05/레벨)

#### HP (체력) - 선택적

- **역할**: 캐릭터 생존 능력. 피격 허용 횟수 결정
- **효과 공식**: `SurvivalTime = HP / (MonsterDPS)`
- **설계 의도**: 높은 스테이지에서 몬스터가 반격할 경우를 대비
- **레벨당 증가량**: `HP(Lv) = BaseHP + (Lv - 1) * 15`
- **비고**: 현 설계에서 몬스터 반격 시스템이 없으면 우선순위 낮음

## 3. 성장 곡선 분석

### 3-1. 몬스터 HP 스케일링

```
MonsterHP(stage) = BaseHP * (1 + stage * 0.15)
```

- **BaseHP** = 50 (일반 몬스터), 200 (정예 몬스터), 500 (구슬 보스)
- 방당 몬스터 수: `min(3 + floor(stage/15), 8)` (최대 8마리)
- 스테이지당 방 수: `min(5 + floor(stage/20), 8)`

### 3-2. 스테이지별 총 HP (스테이지 클리어에 필요한 총 데미지)

가정:
- 방 수: `RoomCount(stage) = min(5 + floor(stage/20), 8)`
- 방당 몬스터 수: `MonstersPerRoom(stage) = min(3 + floor(stage/15), 8)`
- 보스 HP: `BossOrbHP = 500 * (1 + stage * 0.20)`

```
StageTotalHP(stage) = RoomCount * MonstersPerRoom * MonsterHP(stage) + BossHP(stage)
```

### 3-3. 플레이어 DPS vs 몬스터 HP - 적정 성장 곡선

#### "쾌적 클리어" 기준

- **목표 방 클리어 시간**: 3~5초 (몬스터 전투 시간)
- **목표 방 이동 시간**: 1~2초

### 3-4. Sweet Spot 분석

플레이어가 "쾌적하게" 진행하려면 아래 수준의 스탯을 유지해야 한다:

|----------|------------|-------------------|-------------------|------------------|----------|
| 1 | 1 | 1 | 1 | 1 | 11 |
| 10 | 15 | 10 | 5 | 5 | 113 |
| 25 | 25 | 18 | 12 | 10 | 309 |
| 50 | 38 | 26 | 22 | 18 | 642 |
| 100 | 55 | 38 | 35 | 30 | 1,412 |

> **Sweet Spot**: DPS가 필요 DPS의 1.2~2.0배일 때 쾌적하면서도 약간의 도전감이 있다.
> DPS < 필요 DPS * 0.8이면 "벽" 구간으로 느껴지며, 이때 전생(Prestige)을 유도한다.

## 4. DPS 계산 공식

### 4-1. 기본 DPS

```
BaseDPS = ATK * ATK_SPEED
```

- ATK: 1회 타격 피해량
- ATK_SPEED: 초당 타격 횟수
- BaseDPS: 치명타를 고려하지 않은 초당 기본 피해량

### 4-2. 유효 DPS (치명타 포함)

```
EffectiveDPS = BaseDPS * (1 + CRIT_RATE * (CRIT_DMG - 1))
```

- `CRIT_RATE`: 0~0.8 (0%~80%)
- `CRIT_DMG`: 1.5~5.0 (150%~500%)
- `(CRIT_DMG - 1)`: 치명타 시 **추가** 배율 (기본 100% 위에 더해지는 부분)

#### 공식 유도

```
평균 피해 = (1 - CRIT_RATE) * ATK + CRIT_RATE * ATK * CRIT_DMG
          = ATK * [(1 - CRIT_RATE) + CRIT_RATE * CRIT_DMG]
          = ATK * [1 - CRIT_RATE + CRIT_RATE * CRIT_DMG]
          = ATK * [1 + CRIT_RATE * (CRIT_DMG - 1)]

EffectiveDPS = 평균 피해 * ATK_SPEED
             = ATK * ATK_SPEED * [1 + CRIT_RATE * (CRIT_DMG - 1)]
```

### 4-3. 실전 DPS (스킬 포함)

```
ActualDPS = EffectiveDPS * SkillDPSMultiplier
```

- `SkillDPSMultiplier`: 스킬의 추가 DPS 기여 (기본 평타만 = 1.0)
- 스킬이 전체 DPS에서 차지하는 비중은 별도 스킬 설계서에서 정의

### 4-4. 예제 계산

#### 예제 1: 초기 상태 (모든 스탯 레벨 1)

```
ATK = 10, ATK_SPEED = 1.0, CRIT_RATE = 5%, CRIT_DMG = 150%

BaseDPS = 10 * 1.0 = 10.0
CritMultiplier = 1 + 0.05 * (1.5 - 1) = 1 + 0.025 = 1.025
EffectiveDPS = 10.0 * 1.025 = 10.25
```

#### 예제 2: 중반 (스테이지 50 권장 스탯)

```
ATK = 127 (Lv38), ATK_SPEED = 2.25 (Lv26), CRIT_RATE = 21.8% (Lv22), CRIT_DMG = 192.5% (Lv18)

BaseDPS = 127 * 2.25 = 285.75
CritMultiplier = 1 + 0.218 * (1.925 - 1) = 1 + 0.218 * 0.925 = 1.2017
EffectiveDPS = 285.75 * 1.2017 = 343.36
```

> 참고: Sweet Spot 테이블의 642 DPS와 차이가 나는 이유는 스킬 DPS 기여분(약 1.87배)이 포함되기 때문.
> `ActualDPS = 343.36 * 1.87 = 642.08`

#### 예제 3: 후반 (스테이지 200 권장 스탯)

```
ATK = 267 (Lv72), ATK_SPEED = 3.55 (Lv52), CRIT_RATE = 44.2% (Lv50), CRIT_DMG = 260% (Lv45)

BaseDPS = 267 * 3.55 = 947.85
CritMultiplier = 1 + 0.442 * (2.60 - 1) = 1 + 0.442 * 1.60 = 1.7072
EffectiveDPS = 947.85 * 1.7072 = 1,617.86
ActualDPS = 1,617.86 * 1.98 = 3,203.36
```

#### 예제 4: DPS 기여도 비교 (레벨 50 기준)

| 스탯 1레벨 추가 업그레이드 | 변화 전 DPS | 변화 후 DPS | DPS 증가량 | 증가율 |
|---------------------------|------------|------------|-----------|--------|
| ATK (Lv50 → 51) | 343.36 | 356.97 | +13.61 | +3.97% |
| ATK_SPEED (Lv26 → 27) | 343.36 | 350.98 | +7.62 | +2.22% |
| CRIT_RATE (Lv22 → 23) | 343.36 | 345.56 | +2.20 | +0.64% |
| CRIT_DMG (Lv18 → 19) | 343.36 | 345.04 | +1.68 | +0.49% |

## 5. 스탯 효율 분석

### 5-1. 효율 지표 정의

```
StatEfficiency = DPS증가량 / 업그레이드비용(골드)
```

단위: **DPS/골드** (1골드당 얻는 DPS 증가량)

### 5-2. 게임 초반 효율 (스테이지 1~25)

이 시기에는 ATK와 ATK_SPEED의 비용이 저렴하고 DPS 기여도가 높다.

| 스탯 | 현재 레벨 → 목표 | DPS 증가 | 비용 | 효율 (DPS/골드) |
|------|------------------|---------|------|----------------|
| ATK | 10 → 11 | +4.35 | 28 | 0.155 |

## 7. 스탯 상한선 및 소프트캡

### 7-1. 하드캡 (Hard Cap)

절대적 상한선. 이 이상으로 올라갈 수 없다.

| 스탯 | 하드캡 값 | 도달 레벨 | 설계 근거 |
|------|---------|----------|----------|
| CRIT_RATE | 80% | Lv 95 | 100% 치명타는 "크리티컬"의 의미를 상실시킴 |
| CRIT_DMG | 500% | Lv 100 (실질 397.5%) | 5배 이상은 밸런스 파괴 위험 |
| ATK_SPEED | 10.0 | Lv 180+ (실질 불가) | 애니메이션 표현 한계 |
| MOVE_SPEED | 15.0 | Lv 180+ (실질 불가) | 화면 표현 한계 |

### 7-2. 소프트캡 (Soft Cap)

소프트캡 이후 증가량이 절반으로 감소한다.

| 스탯 | 소프트캡 값 | 소프트캡 레벨 | 캡 전 증가량 | 캡 후 증가량 |
|------|-----------|-------------|-------------|-------------|
| ATK | 없음 (선형 증가 유지, 단 Lv51부터 증가량 변화) | 50 | +3/Lv | +5/Lv |
| ATK_SPEED | 5.0 | Lv 81 | +0.05/Lv | +0.025/Lv |
| MOVE_SPEED | 8.0 | Lv 51 | +0.1/Lv | +0.05/Lv |
| CRIT_RATE | 없음 | - | +0.8%/Lv | 동일 |
| CRIT_DMG | 없음 | - | +2.5%/Lv | 동일 |

> **ATK의 특수 처리**: ATK는 소프트캡 대신 Lv51부터 증가량이 +3에서 +5로 **상승**한다.
> 이는 후반부 몬스터 HP 스케일링에 대응하기 위한 의도적 설계이다.

### 7-3. 소프트캡 적용 공식

```csharp
float GetStatValue(float baseValue, int level, float perLevel, float softCapValue, float softCapReduction)
{
    float value = baseValue;
    for (int i = 1; i < level; i++)
    {
        float currentValue = baseValue + (i - 1) * perLevel;
        if (currentValue >= softCapValue)
            value += perLevel * softCapReduction; // 0.5 = 절반
        else
            value += perLevel;
    }
    return Mathf.Min(value, hardCap);
}
```

## 8. ScriptableObject 스키마

### 8-1. CharacterStatsSO

캐릭터의 현재 스탯 값을 관리하는 ScriptableObject.

```csharp
[CreateAssetMenu(fileName = "NewCharacterStats", menuName = "RunningRight/CharacterStats")]
public class CharacterStatsSO : ScriptableObject
{
    [Header("기본 스탯")]
    [Tooltip("기본 공격력")]
    public float ATK = 10f;

    [Tooltip("초당 공격 횟수")]
    public float ATK_SPEED = 1.0f;

    [Tooltip("치명타 확률 (0.0 ~ 0.8)")]
    [Range(0f, 0.8f)]
    public float CRIT_RATE = 0.05f;

    [Tooltip("치명타 배율 (1.0 ~ 5.0)")]
    [Range(1.0f, 5.0f)]
    public float CRIT_DMG = 1.5f;

    [Tooltip("이동 속도 (유닛/초)")]
    public float MOVE_SPEED = 3.0f;

    [Tooltip("최대 체력")]
    public float HP = 100f;

    [Header("스탯 레벨")]
    public int ATK_Level = 1;
    public int ATK_SPEED_Level = 1;
    public int CRIT_RATE_Level = 1;
    public int CRIT_DMG_Level = 1;
    public int MOVE_SPEED_Level = 1;
    public int HP_Level = 1;

    [Header("전생 보너스")]
    [Tooltip("전생 횟수에 따른 전체 스탯 배율")]
    public float PrestigeMultiplier = 1.0f;

    // === 계산 프로퍼티 ===

    /// <summary>
    /// 치명타를 포함한 유효 DPS
    /// </summary>
    public float EffectiveDPS =>
        ATK * ATK_SPEED * (1f + CRIT_RATE * (CRIT_DMG - 1f)) * PrestigeMultiplier;

    /// <summary>
    /// 치명타 미포함 기본 DPS
    /// </summary>
    public float BaseDPS => ATK * ATK_SPEED * PrestigeMultiplier;

    /// <summary>
    /// 공격 간격 (초)
    /// </summary>
    public float AttackInterval => 1f / ATK_SPEED;
}
```

```csharp
    public float GetValueAtLevel(int level)
    {
        float value = BaseValue;
        float increment = ValuePerLevel;

        for (int i = 2; i <= level; i++)
        {
            if (value >= SoftCapValue && ValuePerLevelAfterSoftCap > 0)
                increment = ValuePerLevelAfterSoftCap;
            else
                increment = ValuePerLevel;

            value += increment;
        }

        return Mathf.Min(value, HardCapValue);
    }

    /// <summary>
    /// 지정 레벨로의 업그레이드 비용 계산
    /// </summary>
    public long GetUpgradeCost(int currentLevel)
    {
        if (currentLevel >= MaxLevel) return -1; // 최대 레벨
        return (long)(BaseCost * Mathf.Pow(CostMultiplier, currentLevel - 1));
    }

    /// <summary>
    /// 레벨 1부터 지정 레벨까지의 누적 비용
    /// </summary>
    public long GetTotalCostToLevel(int targetLevel)
    {
        long total = 0;
        for (int i = 1; i < targetLevel; i++)
        {
            total += GetUpgradeCost(i);
        }
        return total;
    }
}
```

## 부록: 전생(Prestige) 시스템과 스탯의 관계

### 전생 보너스 배율

```
PrestigeMultiplier = 1.0 + PrestigeCount * 0.10
```

| 전생 횟수 | 보너스 배율 | 효과 |
|-----------|-----------|------|
| 0 | 1.00x | 기본 |
| 1 | 1.10x | 모든 DPS +10% |
| 2 | 1.20x | 모든 DPS +20% |
| 3 | 1.30x | 모든 DPS +30% |
| 5 | 1.50x | 모든 DPS +50% |
| 10 | 2.00x | 모든 DPS +100% |

### 전생 리셋 범위

| 항목 | 리셋 여부 |
|------|----------|
| 스탯 레벨 | 리셋 (레벨 1로) |
| 스탯 값 | 리셋 (기본값으로) |
| 골드 | 리셋 (0으로) |
| 스테이지 | 리셋 (1로) |
| 전생 횟수 | 유지 (누적) |
| 전생 보너스 배율 | 유지 (영구) |
| 장비 (해금 시) | 유지 |
| 업적 | 유지 |

### 전생 해금 조건

```
첫 번째 전생: Stage 50 도달
이후 전생: 직전 전생 최고 스테이지의 80% 도달
```

> 전생 시 모든 스탯이 리셋되지만 영구 배율이 적용되므로,
> 같은 스탯 레벨에서도 DPS가 더 높아져 이전보다 빠르게 진행할 수 있다.
> 이것이 방치형 게임의 핵심 리텐션 루프이다.
