## 1. UI 설계 원칙

### 1-1. 모바일 퍼스트 (Mobile-First)

- **기준 해상도:** 1920x1080 (16:9)
- **최소 지원:** 1280x720 (16:9)
- **UI 스케일링:** Canvas Scaler - "Scale With Screen Size" 모드, Match = 0.5 (Width/Height 균형)
- 모든 UI 요소는 Canvas 내 앵커(Anchor)와 피벗(Pivot) 기반으로 배치하여 다양한 해상도에 자동 대응

### 1-2. 한 손 조작 (One-Hand Operation)

```
  ┌──────────────────────────────────────────┐
  │                                          │
  │   정보 영역 (읽기 전용)                    │
  │   - 스테이지 정보, HP바, 골드 등            │
  │                                          │
  │                                          │
  │                                          │
  │   게임 뷰 (터치 불필요)                     │
  │   - 자동 전투 화면                         │
  │                                          │
  │                                          │
  │══════════════════════════════════════════│
  │   조작 영역 (엄지 도달 가능)                │  <-- 화면 하단 1/3
  │   - 스킬 버튼, 메뉴 탭, 주요 인터랙션       │
  └──────────────────────────────────────────┘
```

**엄지 존(Thumb Zone) 원칙:**
- 모든 인터랙티브 버튼은 화면 하단 40% 이내에 배치
- 자주 누르는 버튼(스킬, 메뉴 탭)은 하단 25% 이내
- 가끔 누르는 버튼(설정, 자동전투 토글)은 상단에 배치 가능하되 크기를 충분히 확보
- 최소 터치 영역: 48x48dp (약 96x96px @2x)

### 1-3. DNF 패러디 비주얼 테마

- **색상 팔레트:**

| 용도 | 색상 | HEX | 설명 |
|------|------|-----|------|
| 주 배경 | 어두운 남색 | `#1A1A2E` | 던전 분위기 |
| 보조 배경 | 진한 회색 | `#16213E` | 패널/카드 배경 |
| 강조색 (골드) | 금색 | `#FFD700` | 골드, 중요 수치 |
| 강조색 (액션) | 주황색 | `#FF6B35` | 업그레이드 버튼, CTA |
| HP 바 | 빨간색 | `#E63946` | 몬스터/보스 HP |
| MP/쿨타임 | 파란색 | `#4895EF` | 스킬 쿨다운 |
| 긍정 변화 | 초록색 | `#2ECC71` | 스탯 증가, 획득 |
| 부정 변화 | 빨간색 | `#E74C3C` | 비용, 감소 |
| 일반 텍스트 | 흰색 | `#FFFFFF` | 본문 |
| 비활성 텍스트 | 회색 | `#7F8C8D` | 비활성 항목 |

- **폰트:**
  - 제목/숫자: 굵은 고딕 계열 (NotoSansKR-Bold)
  - 본문: 일반 고딕 (NotoSansKR-Regular)
  - 데미지 텍스트: 별도 비트맵 폰트 (DNF 스타일 도트감)

- **UI 스타일:**
  - 패널: 둥근 모서리(radius 12px), 반투명 배경(alpha 85%), 1px 밝은 테두리
  - 버튼: 그라데이션 배경, 눌림 시 스케일 0.95 + 색상 어둡게
  - 아이콘: 32x32 ~ 64x64px, 2px 외곽선

### 1-4. 정보 계층 원칙

| 우선순위 | 정보 | 표현 방식 |
|----------|------|-----------|
| 1 (최고) | 전투 화면 | 화면 중앙, 가장 넓은 영역 |
| 2 | 스킬 버튼 | 하단 고정, 큰 아이콘 + 쿨타임 오버레이 |
| 3 | 골드/재화 | 하단 우측, 항상 표시 |
| 4 | 스테이지/방 정보 | 상단 좌측, 작은 텍스트 |
| 5 | 보스 HP바 | 전투 시에만 상단 중앙 표시 |
| 6 | 설정/자동전투 | 상단 우측, 아이콘만 |

│                                                                 │

### 2-2. 요소별 상세 스펙

#### 스테이지/방 정보 (좌상단)

```
┌─────────────────────┐
│  ◆ STAGE 127        │  ← 폰트: Bold 36px, 색상: #FFD700 (골드)
│    Room 3 / 6       │  ← 폰트: Regular 24px, 색상: #FFFFFF
│  ▓▓▓▓▓▓░░░░  50%   │  ← 방 진행도 바 (너비 200px, 높이 8px)
└─────────────────────┘
  위치: 좌상단 (x:24, y:24)
  크기: 240 x 100 px
  배경: 반투명 검정 (#000000, alpha 60%)
  모서리: 둥근 모서리 (radius 8px)
```

**방 진행도 바:**
- 전체 너비: 200px, 높이: 8px
- 채워진 부분: #FFD700 (골드)
- 빈 부분: #333333
- `(현재 방 - 1) / 총 방 수 * 100%` 로 계산

#### 설정 버튼 (우상단)

```
  ┌──────┐
  │  ⚙   │  ← 아이콘: 48x48px
  │      │
  └──────┘
  위치: 우상단 (right:24, y:24)
  터치 영역: 72x72px (아이콘보다 넓게)
  배경: 반투명 원형 (#000000, alpha 40%)
```

#### 자동전투 토글 (설정 옆)

```
  ┌──────────┐
  │  AUTO    │  ← ON 상태: #2ECC71 배경, "AUTO" 텍스트 Bold 20px
  │  ──●     │  ← 토글 스위치 시각화
  └──────────┘

  ┌──────────┐
  │  AUTO    │  ← OFF 상태: #555555 배경, "AUTO" 텍스트 회색
  │  ●──     │
  └──────────┘
  위치: 설정 버튼 왼쪽 (right:104, y:24)
  크기: 96x48px
  터치 영역: 96x72px
```

**자동전투 ON/OFF 차이:**
- ON (기본): 스킬 자동 시전, 이동 자동
- OFF: 스킬 수동 터치 시전 (이동은 항상 자동)

#### 스킬 버튼 (하단 좌측)

```
  ┌────────────────────────────────────────────────────────────┐
  │                                                            │
  │   ┌──────┐   ┌──────┐   ┌──────┐   ┌──────────┐          │
  │   │ 🗡️   │   │ 🔥   │   │ ⚡   │   │  💥      │          │
  │   │      │   │      │   │      │   │  궁극기  │          │
  │   │ 3.2s │   │ 5.0s │   │ 8.5s │   │  30.0s   │          │
  │   └──────┘   └──────┘   └──────┘   └──────────┘          │
  │   [스킬1]    [스킬2]    [스킬3]    [궁극기]               │
  │                                                            │
  └────────────────────────────────────────────────────────────┘
```

**개별 스킬 버튼 상세:**

```
  ┌────────────────┐
  │  ┌──────────┐  │
  │  │ 스킬     │  │  ← 아이콘: 64x64px
  │  │ 아이콘   │  │
  │  │          │  │
  │  │▓▓▓▓░░░░ │  │  ← 쿨타임 오버레이 (시계방향 sweep, 반투명 검정)
  │  └──────────┘  │
  │   Lv.5         │  ← 스킬 레벨: Regular 16px, #FFFFFF
  │   3.2s         │  ← 남은 쿨타임: Bold 20px, #4895EF
  └────────────────┘
  크기: 일반 스킬 88x110px, 궁극기 108x110px
  간격: 스킬 간 16px
  위치: 하단 좌측 (x:24, bottom:24)
```

**스킬 버튼 상태:**

| 상태 | 시각 표현 |
|------|-----------|
| 사용 가능 | 아이콘 밝게, 쿨타임 오버레이 없음, 미세한 발광(glow) 효과 |
| 쿨타임 중 | 시계방향 sweep 오버레이 (#000000, alpha 70%), 남은 시간 표시 |
| 레벨 미달/잠김 | 아이콘 회색 처리, 자물쇠 아이콘 오버레이 |
| 자동 시전 중 | 테두리 #2ECC71 발광 |
| 수동 터치 시 | 스케일 0.9 → 1.0 바운스 애니메이션 (0.1초) |

#### 골드/재화 표시 (하단 우측)

```
  ┌────────────────┐
  │  💰 1.23M      │  ← 골드: Bold 28px, #FFD700
  │  💎 45         │  ← 보석: Bold 22px, #4895EF
  │  🔮 128        │  ← 구슬조각: Bold 22px, #9B59B6
  └────────────────┘
  위치: 하단 우측 (right:24, bottom:44)
  크기: 200x90px
  배경: 반투명 (#000000, alpha 50%), radius 8px
```

**재화 아이콘 크기:** 24x24px
**재화 획득 시 애니메이션:** 숫자 위로 "+100" 텍스트 페이드아웃 (0.5초)

### 2-3. 전투 영역 HUD 오버레이

전투 중 게임 뷰 위에 표시되는 임시 요소:

```
                      ┌────────────┐
                      │  STAGE     │
                      │  CLEAR!    │  ← 스테이지 클리어 시 1.5초 표시
                      └────────────┘

       -1,234         ← 일반 데미지: Bold 24px, #FFFFFF, 위로 떠오름
       -5,678!        ← 치명타 데미지: Bold 36px, #FFD700, 크게 + 흔들림
       +500           ← 골드 획득: Bold 20px, #FFD700, 위로 떠오름
```

**데미지 텍스트 동작:**
- 몬스터 머리 위에서 생성
- 0.3초간 위로 50px 이동하며 페이드아웃
- 치명타: 1.5배 크기, 0.05초 확대 후 축소 펀치 효과
- 오브젝트 풀링으로 관리 (최대 20개 동시 표시)

## 3. 보스 전투 HUD

### 3-1. 보스 HP 바 등장 조건

마지막 방(Room) 진입 시 개 얼굴 구슬 보스 등장과 함께 보스 HP 바가 화면 상단에 슬라이드인 애니메이션으로 나타남.

### 3-2. 보스 HP 바 와이어프레임

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  Stage 127                                                ⚙ 🔄 │
│                                                                 │
│         ┌───────────────────────────────────────────┐           │
│         │  🐶 개 얼굴 구슬  Lv.127                  │           │
│         │  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░░  72%        │           │
│         │  HP: 245.6K / 341.2K           ⏱ 30s     │           │
│         └───────────────────────────────────────────┘           │
│                                                                 │
│                         🐶                                      │
│        [캐릭터] ──→    (구슬)                                    │
│                                                                 │
│─────────────────────────────────────────────────────────────────│
│  [스킬1] [스킬2] [스킬3] [궁극기]              💰 1.23M        │
└─────────────────────────────────────────────────────────────────┘
```

### 3-3. 보스 HP 바 상세 스펙

```
┌───────────────────────────────────────────────────────┐
│  🐶 개 얼굴 구슬  Lv.127                              │  ← 보스 이름/레벨
│                                                       │
│  ┌─────────────────────────────────────────────────┐  │
│  │▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░░░░░░░░░░░░░│  │  ← HP 바
│  └─────────────────────────────────────────────────┘  │
│                                                       │
│  HP: 245.6K / 341.2K                      ⏱ 30s      │  ← 수치 + 타이머
└───────────────────────────────────────────────────────┘
  위치: 상단 중앙 (center-x, y:100)
  크기: 600 x 100 px
  배경: 반투명 (#0D0D1A, alpha 85%), radius 12px, 테두리 #E63946 1px
```

**HP 바 세부:**
- 전체 너비: 540px, 높이: 24px
- HP 100~60%: #E63946 (빨간색)
- HP 59~30%: #FF6B35 (주황색, 위기감)
- HP 29~0%: #FFD700 (금색, 깜빡임 효과 0.5초 간격)
- HP 감소 시: 기존 HP 바 위에 #FFFFFF 잔상이 0.3초간 천천히 줄어듦 (래깅 효과)

**타이머 (선택 사항):**
- 보스 전투 제한 시간: 30초 (기본, 스테이지에 따라 조절 가능)
- 10초 이하: 빨간색 + 깜빡임
- 시간 초과 시: 보스 HP 회복 또는 이전 방으로 강제 이동
- 타이머 미적용 시: 해당 영역에 DPS 표시 가능

### 3-4. 보스 등장/처치 연출

**등장 연출 (1초):**
1. 화면 양 옆에서 검은 레터박스(cinematic bar) 슬라이드인 (0.3초)
2. "WARNING!" 텍스트 깜빡임 (0.5초, Bold 72px, #E63946)
3. 보스 HP 바 상단에서 슬라이드 다운 (0.2초)
4. 레터박스 페이드아웃

**처치 연출 (1.5초):**
1. 구슬 깨지는 파티클 이펙트 (0.5초)
2. 화면 화이트 플래시 (0.1초)
3. 카메라 셰이크 (0.3초)
4. "STAGE CLEAR!" 텍스트 중앙 표시 (Bold 64px, #FFD700, 1초)
5. 보스 HP 바 페이드아웃
6. 보상 획득 텍스트 표시

## 4. 메인 메뉴(로비) 화면

### 4-1. 하단 탭 내비게이션

인게임 HUD 상태에서 하단 탭을 통해 메뉴에 접근한다. 메뉴는 게임 뷰 위에 오버레이 형태로 표시되며, 배경의 전투는 계속 진행된다.

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│                                                         [X 닫기]│
│                                                                 │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │                                                           │  │
│  │                                                           │  │
│  │              탭 콘텐츠 영역                                │  │
│  │              (각 탭에 따라 변경)                            │  │
│  │                                                           │  │
│  │              높이: 화면의 약 65%                            │  │
│  │                                                           │  │
│  │                                                           │  │
│  │                                                           │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                 │

### 9-5. 포맷팅 구현 참조

```
포맷팅 우선순위 결정 로직:

값 < 1,000          → 그대로 표시
값 < 1,000,000      → 콤마 구분 (예: 999,999)
값 < 1,000,000,000  → / 1,000,000 + "M" (예: 1.23M)
값 < 1e12           → / 1,000,000,000 + "B" (예: 1.23B)
값 < 1e15           → / 1e12 + "T" (예: 1.23T)
값 >= 1e15          → 알파벳 단위 순환 (aa, ab, ac, ... az, ba, ...)

소수점 처리:
  약식 값 < 10   → 소수점 2자리 (예: 1.23M)
  약식 값 < 100  → 소수점 2자리 (예: 12.34M)
  약식 값 >= 100 → 소수점 1자리 (예: 123.4M)

특수 상황:
  - 강화 비용 표시: 약식 사용 가능 (💰 1.23M)
  - 데미지 텍스트: 약식 사용 (화면 공간 절약)
  - 상세 팝업 내: 풀 표기 + 약식 병행 (예: "245,600 (245.6K)")
  - 변동량 표시: 항상 부호 표시 (+75, -30)
```

## 11. Safe Area 대응

### 11-1. Safe Area 개요

Android 기기의 노치, 홀 펀치 카메라, 둥근 모서리, 내비게이션 바 등에 의해 가려지는 영역을 피하여 UI를 배치한다.

│ ░  ┌──── 노치/상태바 영역 ────┐                              ░ │
│ ░  └──────────────────────────┘                              ░ │
│ ░                                                            ░ │
│ ░  ┌──────────────────────────────────────────────────────┐  ░ │
│ ░  │                                                      │  ░ │
│ ░  │              S A F E   A R E A                       │  ░ │
│ ░  │              (모든 UI 콘텐츠는 이 안에 배치)          │  ░ │
│ ░  │                                                      │  ░ │

└─────────────────────────────────────────────────────────────────┘
  ░ = 둥근 모서리로 인한 위험 영역
```

### 11-2. Unity 구현 전략

**Canvas 설정:**
```
Canvas
├── Canvas Scaler
│   ├── UI Scale Mode: Scale With Screen Size
│   ├── Reference Resolution: 1920 x 1080
│   ├── Screen Match Mode: Match Width Or Height
│   └── Match: 0.5
└── Safe Area Panel (모든 UI의 부모)
    ├── RectTransform: Safe Area에 맞게 동적 조절
    └── SafeAreaAdapter.cs 컴포넌트 부착
```

**SafeAreaAdapter 동작 원리:**
```
1. Screen.safeArea 값을 매 프레임(또는 해상도 변경 시) 체크
2. RectTransform의 anchorMin, anchorMax를 Safe Area에 맞게 조절
3. 모든 UI 자식 요소는 이 패널 내부에 배치되므로 자동으로 Safe Area 내에 위치
```

### 11-3. 영역별 Safe Area 처리

| UI 요소 | Safe Area 적용 | 비고 |
|---------|---------------|------|
| 스테이지 정보 (좌상단) | 적용 | 노치에 가려지지 않도록 패딩 |
| 설정/자동전투 (우상단) | 적용 | 펀치홀 카메라 회피 |
| 스킬 버튼 (하단) | 적용 | 내비게이션 바 위에 배치 |
| 하단 탭 바 | 적용 | 내비게이션 바 위에 배치, 제스처 바와 겹치지 않도록 |
| 게임 뷰 (전투 화면) | 미적용 | 풀스크린 렌더링 (배경은 화면 전체) |
| 팝업 배경 딤 | 미적용 | 화면 전체 커버 |
| 팝업 콘텐츠 | 적용 | Safe Area 내 중앙 배치 |
| 보스 HP 바 | 적용 | 상단 Safe Area 내 |

### 9-slice 설정

모든 9-slice 에셋은 모서리 고정 + 중앙 스트레치 방식:
- 모서리 영역: 16~24px (에셋에 따라)
- 패딩: 에셋 크기의 25% 이내
