## 1. 전생 시스템 개요

### 1-1. 핵심 개념

전생(Prestige)은 플레이어가 현재 진행 상태를 자발적으로 초기화하고, 그 대가로 **영구적인 강화 효과**를 얻는 시스템이다. 방치형 게임의 핵심 리텐션 루프로, 플레이어에게 다음과 같은 동기를 부여한다:

- **성장 천장 돌파**: 현재 강화만으로는 넘기 어려운 스테이지 벽을 전생 버프로 극복
- **더 빠른 진행**: 전생 후 초반 구간을 압도적으로 빠르게 클리어하는 쾌감
- **장기 목표 설정**: 전생 포인트 누적 → 상점 업그레이드 → 더 높은 스테이지 도달의 순환

### 1-2. 전생 흐름 요약

```
일반 진행 (스테이지 1 → N)
    ↓ 성장 정체 (몬스터 HP 스케일링 > 플레이어 DPS)
    ↓ 전생 해금 조건 충족
    ↓ [전생 실행]
    ↓ 전생 포인트 획득 (도달 스테이지 기반)
    ↓ 영구 버프 상점에서 강화 구매
    ↓ 스테이지 1로 리셋 (영구 버프 적용 상태)
    ↓ 이전보다 빠른 진행 → 더 높은 스테이지 도달
    ↓ 다시 전생 → 반복
```

### 1-3. 왜 전생을 하는가?

| 동기 | 설명 |
|------|------|
| 스테이지 벽 | 몬스터 HP = `BaseHP * (1 + stage * 0.15)` → 스테이지가 높아질수록 기하급수적 난이도 상승 |
| 골드 수입 한계 | 스테이지 진행이 느려지면 시간당 골드 수입도 정체 |
| 영구 버프 유혹 | 전생 포인트로 구매한 버프는 영원히 유지 → 누적될수록 강력 |
| 속도 쾌감 | 전생 후 초반 구간을 순삭하는 경험이 중독적 |

## 2. 전생 해금 조건

### 2-1. 첫 전생 해금

- **최소 도달 스테이지: 50**
- 스테이지 50 도달 시 메인 HUD에 "전생" 버튼 활성화
- 버튼 옆에 획득 예정 전생 포인트 미리보기 표시

### 2-2. 이후 전생 조건

전생은 **언제든 반복 가능**하되, 최소 조건이 존재한다:

| 전생 횟수 | 최소 도달 스테이지 |
|-----------|-------------------|
| 1회차 | 50 |
| 2회차 | 50 |
| 3회차 | 50 |
| N회차 | 50 (고정) |

> **설계 의도**: 최소 스테이지를 고정(50)으로 유지하여, 초반에 빠르게 전생을 반복하며 포인트를 쌓는 "속도 전생" 전략도 허용한다. 대신, 낮은 스테이지에서 전생하면 포인트 획득이 극히 적어 비효율적이므로 자연스럽게 억제된다.

```
│                                      │

└──────────────────────────────────────┘
```

## 3. 전생 보상 체계

### 3-1. 전생 포인트 (전생석)

전생 시 획득하는 영구 재화. 이름: **"전생석"** (Rebirth Stone)

#### 획득 공식

```
전생석 = floor( (maxStage - 49)^1.3 * 0.5 )
```

- `maxStage`: 해당 런에서 도달한 최고 스테이지
- 스테이지 50 미만에서는 전생 불가 → 최소 1포인트 보장

#### 스테이지별 전생석 획득 테이블

| 도달 스테이지 | (maxStage - 49) | 전생석 획득 |
|-------------|-----------------|------------|
| 50 | 1 | 0 (최소 보장 → **1**) |
| 55 | 6 | 4 |
| 60 | 11 | 9 |
| 70 | 21 | 20 |
| 80 | 31 | 33 |
| 90 | 41 | 48 |
| 100 | 51 | 65 |
| 120 | 71 | 100 |
| 150 | 101 | 157 |
| 200 | 151 | 261 |
| 300 | 251 | 497 |
| 500 | 451 | 1044 |

> **설계 의도**: 지수 1.3을 사용하여 높은 스테이지 도달 시 보상이 가속적으로 증가. "조금만 더 버티면 훨씬 많은 포인트"라는 동기를 부여한다.

### 3-2. 전생 보너스

연속 전생 또는 특정 이정표 달성 시 추가 보너스:

| 조건 | 보너스 |
|------|--------|
| 최초 전생 | +10 전생석 보너스 |
| 전생 5회 누적 | +20 전생석 보너스 (1회성) |
| 전생 10회 누적 | +50 전생석 보너스 (1회성) |
| 전생 20회 누적 | +100 전생석 보너스 (1회성) |
| 신기록 스테이지 달성 | 전생석 +20% 추가 |

## 4. 영구 버프 상점

### 4-1. 상점 개요

전생석으로 구매하는 영구 업그레이드. 전생을 해도 절대 초기화되지 않는다.

### 4-2. 업그레이드 목록

#### 전투 계열 (6종)

| # | 이름 | 설명 | 최대 레벨 | 레벨당 비용 | 레벨당 효과 |
|---|------|------|----------|------------|------------|
| 1 | **영혼의 힘** | 영구 공격력 증가 | 50 | `10 + (lv * 5)` | ATK +5% |
| 2 | **날카로운 본능** | 영구 치명타 확률 증가 | 20 | `15 + (lv * 8)` | CRIT_RATE +1% |
| 3 | **파멸의 일격** | 영구 치명타 데미지 증가 | 30 | `12 + (lv * 6)` | CRIT_DMG +5% |
| 4 | **전투의 광기** | 영구 공격 속도 증가 | 25 | `12 + (lv * 7)` | ATK_SPEED +3% |
| 5 | **폭풍의 발걸음** | 영구 이동 속도 증가 | 15 | `8 + (lv * 4)` | MOVE_SPEED +4% |
| 6 | **스킬 마스터리** | 스킬 시작 레벨 증가 | 10 | `20 + (lv * 15)` | 스킬 시작 Lv +1 |

#### 경제 계열 (4종)

| # | 이름 | 설명 | 최대 레벨 | 레벨당 비용 | 레벨당 효과 |
|---|------|------|----------|------------|------------|
| 7 | **황금 손** | 영구 골드 획득량 증가 | 30 | `8 + (lv * 4)` | 골드 획득 +10% |
| 8 | **구슬 사냥꾼** | 영구 구슬 조각 획득량 증가 | 20 | `12 + (lv * 6)` | 구슬 조각 획득 +8% |
| 9 | **할인의 달인** | 스탯 강화 비용 감소 | 15 | `15 + (lv * 8)` | 강화 비용 -3% |
| 10 | **전생석 연금술** | 전생 시 전생석 추가 획득 | 10 | `25 + (lv * 20)` | 전생석 획득 +5% |

#### 편의 계열 (4종)

| # | 이름 | 설명 | 최대 레벨 | 레벨당 비용 | 레벨당 효과 |
|---|------|------|----------|------------|------------|
| 11 | **시작 가속** | 전생 후 시작 스테이지 증가 | 10 | `30 + (lv * 25)` | 시작 스테이지 +3 |
| 12 | **부활의 유산** | 전생 후 초기 골드 보유 | 15 | `10 + (lv * 5)` | 시작 골드 +500 * lv |
| 13 | **오프라인 수행** | 오프라인 보상 효율 증가 | 20 | `10 + (lv * 6)` | 오프라인 효율 +3% |
| 14 | **자동 강화** | 전생 후 자동 스탯 강화 | 5 | `40 + (lv * 30)` | 최적 배분 자동 강화 lv * 5회 |

### 4-3. 비용 공식 상세

각 업그레이드의 레벨 L에서의 비용:

```
Cost(L) = BaseCost + (L * CostPerLevel)
```

누적 비용 (레벨 1 → N까지의 총 비용):

```
TotalCost(N) = Σ(L=0 to N-1) [BaseCost + (L * CostPerLevel)]
             = N * BaseCost + CostPerLevel * N * (N-1) / 2
```

#### 예시: "영혼의 힘" 누적 비용

| 목표 레벨 | 해당 레벨 비용 | 누적 비용 |
|----------|---------------|----------|
| 1 | 10 | 10 |
| 5 | 30 | 100 |
| 10 | 55 | 325 |
| 20 | 105 | 1,150 |
| 30 | 155 | 2,475 |
| 40 | 205 | 4,300 |
| 50 | 255 | 6,625 |

### 4-4. 업그레이드 효과 적용 방식

영구 버프는 **곱연산** 방식으로 기본 스탯에 적용:

```csharp
float finalATK = baseATK * (1 + upgradeATK + statATK);
// upgradeATK: 전생 버프 총합 (예: 영혼의 힘 10레벨 → 0.50)
// statATK: 런 내 골드 강화 총합

float finalGold = baseGold * (1 + goldBonus);
// goldBonus: 황금 손 총합 (예: 10레벨 → 1.00)
```

### 4-5. 추천 업그레이드 우선순위

```
초반 (전생 1~3회):
  1순위: 영혼의 힘 (ATK) → 스테이지 진행 속도 직결
  2순위: 황금 손 (골드) → 런 내 강화 가속
  3순위: 시작 가속 → 초반 구간 스킵

중반 (전생 4~10회):
  1순위: 전생석 연금술 → 포인트 수입 가속
  2순위: 날카로운 본능 + 파멸의 일격 → DPS 폭발
  3순위: 전투의 광기 → 클리어 속도

후반 (전생 11회+):
  1순위: 스킬 마스터리 → 고레벨 스킬로 빠른 진행
  2순위: 시작 가속 고레벨 → 초반 30스테이지 스킵
  3순위: 잔여 포인트로 밸런스 배분
```

## 6. 전생 효율 곡선

### 6-1. 핵심 트레이드오프

**"더 오래 파밍 vs 지금 전생"**

- 더 진행하면: 더 많은 전생석 획득 (고스테이지 보상 가속)
- 지금 전생하면: 영구 버프 적용 후 더 빠른 다음 런

### 6-2. 시간당 전생석 효율 분석

전생 효율의 핵심 지표: **시간당 전생석 획득량**

```
효율 = 전생석_획득 / 런_소요시간(분)
```

#### 전생 0회 기준 (영구 버프 없음) 효율 테이블

| 도달 스테이지 | 전생석 | 소요 시간(분) | 시간당 효율 |
|-------------|--------|-------------|-----------|
| 50 | 1 | 180 | 0.006 |
| 55 | 4 | 200 | 0.020 |
| 60 | 9 | 220 | 0.041 |
| 70 | 20 | 260 | 0.077 |
| 80 | 33 | 310 | 0.106 |
| 90 | 48 | 380 | 0.126 |
| 100 | 65 | 480 | **0.135** |
| 110 | 85 | 600 | 0.142 |
| 120 | 100 | 780 | 0.128 ↓ |

> **최적 전생 시점**: 효율이 하락하기 시작하는 지점 직전. 첫 런 기준 **스테이지 100~110** 부근이 이론적 최적이나, 실제로는 50~60에서 전생하여 빠르게 영구 버프를 확보하는 것이 장기적으로 유리하다.

### 6-3. 전생 효율 곡선 그래프 (개념도)

```

전생석  │
효율    │                    ╭──── 전생 5회+ (영구 버프 누적)
        │                 ╭─╯
        │              ╭──╯
        │           ╭──╯
```

### 6-4. 실전 전생 가이드라인

| 상황 | 권장 전생 시점 |
|------|-------------|
| 최초 전생 | 스테이지 50~55 (빠르게 영구 버프 확보) |
| 전생 2~5회 | 진행이 10분 이상 멈추는 스테이지 |
| 전생 6~15회 | 현재 최고 기록 근처에서 3분 이상 정체 시 |
| 전생 16회+ | 기록 갱신보다 포인트 파밍 목적이면 효율 최적점에서 |
| 신기록 도전 | 가능한 한 버티기 (20% 보너스 획득) |

### 6-5. 조기 전생 vs 지연 전생 시뮬레이션

2시간 플레이 기준 시뮬레이션 (전생 3회 상태):

**전략 A: 조기 전생 (스테이지 100에서 전생, 30분/런)**
```
런 1: 100 → 전생석 65
런 2: 105 → 전생석 70
런 3: 108 → 전생석 74
런 4: 110 → 전생석 77
합계: 4런 × 30분 = 120분, 전생석 286
```

**전략 B: 지연 전생 (스테이지 150까지 파밍, 90분/런)**
```
런 1: 150 → 전생석 157
런 2: (시간 부족, 30분 진행) 80 → 전생석 33
합계: 1.3런, 전생석 190
```

**결론**: 조기 전생(전략 A)이 같은 시간 대비 ~50% 더 많은 전생석을 획득. 단, 스테이지 기록 갱신 시 20% 보너스가 있으므로, 기록 근처에서는 한 번 더 밀어보는 것이 유리.

### 8-4. 리셋 처리 코드 구조 (설계 참고)

```csharp
public class PrestigeManager
{
    public PrestigeResult ExecutePrestige()
    {
        // 1. 전생석 계산
        int points = CalculatePrestigePoints(currentMaxStage);

        // 2. 보너스 적용
        points = ApplyBonuses(points); // 신기록, 연금술 등

        // 3. 전생석 지급
        permanentData.rebirthStones += points;
        permanentData.rebirthCount++;

        // 4. 리셋 항목 초기화
        ResetProgress();    // 스테이지, 방
        ResetCurrency();    // 골드, 구슬 조각
        ResetUpgrades();    // 스탯 강화, 스킬 레벨
        UnequipAll();       // 장비 해제 (보존은 유지)

        // 5. 조건부 유지 적용
        ApplyStartingStage();   // 시작 가속
        ApplyStartingGold();    // 부활의 유산
        ApplyStartingSkills();  // 스킬 마스터리
        ApplyAutoUpgrade();     // 자동 강화

        // 6. 저장
        SavePermanentData();
        SaveRunData();

        return new PrestigeResult(points, permanentData);
    }

    private int CalculatePrestigePoints(int maxStage)
    {
        if (maxStage < 50) return 0;
        int raw = (int)Math.Floor(Math.Pow(maxStage - 49, 1.3) * 0.5);
        return Math.Max(raw, 1);
    }
}
```

## 9. 다중 전생 계층 (각성 시스템)

### 9-1. 각성 개요

전생을 충분히 반복한 후반 플레이어를 위한 **2차 전생 계층**. 전생이 "1차 초기화"라면 각성은 "메타 초기화"에 해당한다.

- 이름: **"각성"** (Awakening)
- 해금 조건: **전생 20회 달성 + 스테이지 300 도달**
- 각성 실행 시: 전생 상점 업그레이드 일부 초기화 → 더 강력한 "각성 버프" 획득

### 9-2. 각성 시 리셋 범위

| 항목 | 각성 시 |
|------|--------|
| 전생 횟수 | 0으로 리셋 |
| 전생 상점 업그레이드 | **전부 초기화** (전생석 환불 없음) |
| 전생석 | 0으로 리셋 |
| 보석 | **유지** |
| 각성 포인트 | 획득 (새로운 영구 재화) |
| 각성 버프 | 유지 |
| 업적 | 유지 |
| 장비 | 유지 |

### 9-3. 각성 포인트 획득 공식

```
각성포인트 = floor( (totalRebirthCount * highestStage) / 500 )
```

| 전생 횟수 | 최고 스테이지 | 각성 포인트 |
|-----------|-------------|------------|
| 20 | 300 | 12 |
| 25 | 400 | 20 |
| 30 | 500 | 30 |
| 40 | 600 | 48 |

### 9-4. 각성 버프 상점

각성 포인트로 구매하는 메타 업그레이드. 전생 버프보다 훨씬 강력하다.

| # | 이름 | 설명 | 최대 레벨 | 비용/Lv | 효과/Lv |
|---|------|------|----------|---------|---------|
| 1 | **존재의 초월** | 모든 데미지 배율 | 10 | 5 + (lv * 3) | 전체 데미지 +20% |
| 2 | **시간의 지배자** | 전생 효율 가속 | 5 | 8 + (lv * 5) | 전생석 획득 +30% |
| 3 | **황금 시대** | 모든 재화 획득 배율 | 8 | 6 + (lv * 4) | 모든 재화 +25% |
| 4 | **영겁의 기억** | 전생 시 전생석 잔존 | 5 | 10 + (lv * 8) | 전생석 리셋 시 lv * 10% 보존 |
| 5 | **차원 도약** | 전생 후 시작 위치 대폭 상승 | 5 | 12 + (lv * 10) | 시작 스테이지 +10 (시작 가속과 별도) |
| 6 | **무한의 재능** | 전생 상점 업그레이드 상한 증가 | 3 | 15 + (lv * 12) | 모든 전생 버프 최대 레벨 +5 |

### 9-5. 각성 시스템 진행 로드맵

```
Phase 1: 전생 루프 (전생 1~19회)
  │  전생석 누적 → 영구 버프 강화 → 더 높은 스테이지
  │
  ▼
Phase 2: 첫 각성 (전생 20회, 스테이지 300)
  │  각성 포인트 ~12 획득
  │  전생 상점 리셋, 처음부터 다시 전생 루프
  │  단, 각성 버프(예: 전체 데미지 +20%)가 적용된 상태
  │
  ▼
Phase 3: 각성 후 빠른 전생 루프 (전생 1~25회, 2차)
  │  각성 버프 덕분에 전생 속도 대폭 단축
  │  더 높은 스테이지(400+) 도달
  │
  ▼
Phase 4: 2차 각성 (전생 30회+, 스테이지 500)
  │  각성 포인트 ~30 추가 획득
  │  더 강력한 각성 버프 구매
  │
  ▼
Phase 5: 반복... (이론적 무한)
```

### 9-6. 각성 해금 조건 요약

| 조건 | 값 | 비고 |
|------|---|------|
| 최소 전생 횟수 | 20회 | 누적 |
| 최소 도달 스테이지 | 300 | 해당 각성 직전 런에서 |
| 각성 UI 노출 시점 | 전생 15회 | 미리보기로 동기 부여 |

### 9-7. 각성 연출

각성은 전생보다 더 화려한 연출:

```
[Phase 1: 0.0s ~ 1.5s] 세계 붕괴
  - 배경이 조각조각 깨지는 연출
  - 캐릭터가 빛에 감싸임
  - 화면 진동 + 저주파 사운드

[Phase 2: 1.5s ~ 3.5s] 각성 변신
  - 캐릭터 실루엣 변화 (아우라 추가)
  - 전생석이 모여 "각성의 결정" 형성
  - 각성 포인트 카운트업 + 화려한 파티클
  - 전용 BGM 인트로 시작

[Phase 3: 3.5s ~ 5.0s] 새로운 시작
  - "새로운 차원의 힘이 깨어났습니다"
  - 각성 레벨 표시 (각성 1 → 각성 2 등)
  - 보상 요약 화면으로 전환
```

### C. ScriptableObject 데이터 구조 (구현 참고)

```csharp
[CreateAssetMenu(fileName = "PrestigeUpgrade", menuName = "Data/PrestigeUpgrade")]
public class PrestigeUpgradeSO : ScriptableObject
{
    public string upgradeName;        // 업그레이드 이름
    public string description;        // 설명
    public UpgradeCategory category;  // 전투/경제/편의
    public int maxLevel;              // 최대 레벨
    public int baseCost;              // 기본 비용
    public int costPerLevel;          // 레벨당 비용 증가
    public float effectPerLevel;      // 레벨당 효과 수치
    public StatType targetStat;       // 적용 대상 스탯
    public Sprite icon;               // 아이콘
}

public enum UpgradeCategory { Combat, Economy, Utility }
public enum StatType { ATK, CRIT_RATE, CRIT_DMG, ATK_SPEED, MOVE_SPEED,
                       SkillLevel, GoldGain, OrbGain, UpgradeCost,
                       RebirthBonus, StartStage, StartGold,
                       OfflineEfficiency, AutoUpgrade }
```
