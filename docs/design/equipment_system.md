## 1. 장비 시스템 개요

### 1-1. 설계 철학

방치형 게임의 특성상 장비 시스템은 **간결하되 보상감이 큰** 구조로 설계한다.
플레이어가 복잡한 의사결정 없이도 성장을 느낄 수 있으면서, 가끔 드롭되는
고등급 장비가 "와!" 하는 순간을 만들어야 한다.

- DNF 패러디답게 장비 이름과 효과를 유머러스하게 구성
- 자동 장착 기능으로 방치 플레이 흐름을 끊지 않음
- 구슬 조각(Orb Fragment) 기반 가챠로 수집 재미 제공

### 1-2. 장비 슬롯 구성

총 **5개 슬롯**을 운용한다.

| 슬롯 | EquipSlot (enum) | 주요 영향 스탯 | 설명 |
|------|-------------------|----------------|------|
| 무기 | `Weapon` | ATK, CRIT_DMG | 공격의 핵심. 가장 큰 ATK 보너스 |
| 상의 | `Armor` | HP, ATK | 방어와 약간의 공격력 |
| 하의 | `Pants` | HP, MOVE_SPEED | 체력과 이동 속도 |
| 어깨 | `Shoulder` | ATK, ATK_SPEED | 공격력과 공격 속도 |
| 악세서리 | `Accessory` | CRIT_RATE, CRIT_DMG | 치명타 특화 |

### 1-3. 스탯 적용 공식

장비 스탯은 캐릭터 기본 스탯에 **가산(additive)** 방식으로 적용된다.

```
finalATK = baseATK + Sum(equipATK) + Sum(setEffectATK) + enhanceBonusATK
finalCRIT_RATE = baseCRIT_RATE + Sum(equipCRIT_RATE)  // 상한 80%
finalCRIT_DMG = baseCRIT_DMG + Sum(equipCRIT_DMG)
finalATK_SPEED = baseATK_SPEED + Sum(equipATK_SPEED)  // 상한 300%
finalMOVE_SPEED = baseMOVE_SPEED + Sum(equipMOVE_SPEED)
```

스탯 상한(Cap):
- `CRIT_RATE`: 최대 80%
- `ATK_SPEED`: 최대 300% (기본 100% 기준)
- `MOVE_SPEED`: 최대 200% (기본 100% 기준)

## 2. 장비 등급 체계

### 2-1. 등급 정의

| 등급 | Rarity (enum) | 색상 코드 | 스탯 배율 범위 | 특수 효과 |
|------|---------------|-----------|---------------|----------|
| 일반 | `Common` | `#FFFFFF` (흰색) | 1.0x | 없음 |
| 고급 | `Uncommon` | `#4CAF50` (초록) | 1.3x ~ 1.5x | 없음 |
| 희귀 | `Rare` | `#2196F3` (파랑) | 1.6x ~ 2.0x | 없음 |
| 영웅 | `Epic` | `#9C27B0` (보라) | 2.2x ~ 2.8x | 랜덤 부가 옵션 1개 |
| 전설 | `Legendary` | `#FF9800` (주황) | 3.0x ~ 4.0x | 고유 특수 효과 + 부가 옵션 2개 |
| 신화 | `Mythic` | `#FF5722` (주황+빛줄기) | 4.5x ~ 6.0x | 고유 스킬 부여 + 부가 옵션 3개 |

### 2-2. 스탯 배율 적용 방식

장비의 최종 스탯은 다음과 같이 계산된다:

```
equipStat = BaseStat * Random(minMultiplier, maxMultiplier)
```

예시: `Common` 무기의 BaseStat ATK가 10이라면, 최종 ATK = `10 * 1.0 = 10`
예시: `Epic` 무기의 BaseStat ATK가 10이라면, 최종 ATK = `10 * Random(2.2, 2.8)` = 22~28

### 2-3. 부가 옵션 풀

`Epic` 이상 등급에서 부여되는 랜덤 부가 옵션:

| 부가 옵션 | 수치 범위 | 비고 |
|-----------|----------|------|
| ATK +N | +5 ~ +30 | |
| ATK_SPEED +N% | +3% ~ +15% | |
| CRIT_RATE +N% | +2% ~ +10% | |
| CRIT_DMG +N% | +5% ~ +25% | |
| MOVE_SPEED +N% | +3% ~ +12% | |
| 골드 획득량 +N% | +5% ~ +20% | 방치형 특화 |
| 구슬 조각 획득량 +N% | +3% ~ +15% | 방치형 특화 |

## 4. 장비 획득 방법

장비 획득은 크게 두 가지 경로로 나뉜다:
- **주 경로 (Primary)**: 보스 구슬 파괴를 통한 직접 드랍 → [5. 구슬 등급 & 장비 드랍 시스템](#5-구슬-등급--장비-드랍-시스템) 참조
- **보조 경로 (Secondary)**: 구슬 조각 가챠 및 일반/정예 몬스터 드랍

> **설계 의도**: 보스 구슬 파괴가 핵심 보상 루프이므로 최고 등급 장비는 구슬 드랍으로만 획득 가능하다.
> 가챠는 구슬 조각을 축적하여 추가 장비를 확보하는 **보충 수단**이다.

### 4-1. 일반/정예 몬스터 드롭 (보조)

몬스터를 처치하면 일정 확률로 장비가 드롭된다. 보스 구슬 드랍에 비해 등급 상한이 낮다.

**몬스터 유형별 드롭률:**

| 몬스터 유형 | 장비 드롭 확률 | 최대 드롭 등급 |
|------------|--------------|---------------|
| 일반 몬스터 | 1.0% | Rare |
| 정예 몬스터 | 5.0% | Epic |

**등급별 드롭 확률 (장비가 드롭되었을 때의 조건부 확률):**

| 등급 | 일반 몬스터 | 정예 몬스터 |
|------|-----------|-----------|
| Common | 60% | 35% |
| Uncommon | 30% | 35% |
| Rare | 10% | 20% |
| Epic | - | 10% |

**슬롯 결정**: 드롭 시 5개 슬롯 중 균등 확률(각 20%)로 결정.

### 4-2. 구슬 조각 가챠 (보조)

개 얼굴 구슬 파괴 시 획득하는 **구슬 조각(Orb Fragment)**으로 가챠를 돌린다.
가챠는 보스 구슬 드랍의 **보충 수단**으로, Legendary까지만 등장한다 (Mythic은 구슬 직접 드랍 전용).

**가챠 비용:**

| 가챠 유형 | 비용 | 설명 |
|----------|------|------|
| 단일 뽑기 | 구슬 조각 50개 | 장비 1개 획득 |
| 10연 뽑기 | 구슬 조각 450개 (10% 할인) | 장비 10개 획득, **Rare 이상 1개 보장** |

**가챠 등급 확률:**

| 등급 | 확률 |
|------|------|
| Common | 40% |
| Uncommon | 35% |
| Rare | 18% |
| Epic | 6% |
| Legendary | 1% |

> **참고**: 가챠 천장 시스템은 기존과 동일하게 유지된다 (소천장 50회/대천장 100회).
> 보스 구슬의 Pity 시스템은 별도로 운영된다 — [5-4. Pity 시스템](#5-4-pity-시스템-소프트--하드) 참조.

### 4-3. 가챠 천장(Pity) 시스템

플레이어의 불만을 방지하고 공정한 보상감을 제공하기 위한 안전장치.

| 천장 유형 | 조건 | 보장 내용 |
|----------|------|----------|
| 소천장 (Soft Pity) | 50회 연속 Epic 이상 미획득 | 51회째부터 Epic 확률 2배 (+6% = 12%) |
| 대천장 (Hard Pity) | 100회 연속 Legendary 미획득 | 100회째 **Legendary 확정** |

- 천장 카운터는 해당 등급 장비 획득 시 리셋
- 가챠 기록(pityCounter)은 저장 데이터에 포함

### 4-4. 구슬 조각 획득량 기준

| 소스 | 획득량 |
|------|--------|
| 보스 구슬 파괴 (기본) | `5 * (1 + floor(stage/10) * 0.25)` |
| 오프라인 보상 | 시간당 (stage * 0.5)개, 효율 계수 0.3 적용 |
| 업적 보상 | 고정 보상 (업적별 상이) |

**스테이지별 구슬 조각 획득 예시:**

| 스테이지 | 보스 1회 획득량 | 단일 뽑기(50개)까지 | 비고 |
|----------|----------------|-------------------|------|
| 1~9 | 5개 | 보스 10회 | `5 * (1 + 0) = 5` |
| 10~19 | 6개 (6.25 → 내림) | 보스 약 8회 | `5 * (1 + 1 * 0.25) = 6.25` |
| 20~29 | 7개 (7.5 → 내림) | 보스 약 7회 | `5 * (1 + 2 * 0.25) = 7.5` |
| 50~59 | 11개 (11.25 → 내림) | 보스 약 5회 | `5 * (1 + 5 * 0.25) = 11.25` |
| 100~109 | 17개 (17.5 → 내림) | 보스 약 3회 | `5 * (1 + 10 * 0.25) = 17.5` |

**전설 등급 기대 획득 소요 (대천장 100회 = 구슬 조각 5,000개 기준):**

| 스테이지 | 보스 1회 획득량 | 대천장까지 보스 횟수 |
|----------|----------------|-------------------|
| 1 | 5개 | 1,000회 |
| 10 | 6개 | 약 834회 |
| 50 | 11개 | 약 455회 |
| 100 | 17개 | 약 294회 |

## 5. 구슬 등급 & 장비 드랍 시스템

> **이 섹션은 장비 획득의 핵심 경로를 정의한다.**
> 보스 구슬(개 얼굴 구슬) 파괴 시 구슬의 등급에 따라 장비가 직접 드랍되며,
> 구슬의 색상 변화 연출이 플레이어에게 도파민과 기대감을 제공하는 핵심 메커니즘이다.
> (원본: `01_core_loop.md` 섹션 3-2, 3-2a, 3-2b, 3-3)

### 5-1. 구슬 등급 & 색상 테이블

구슬을 타격할 때마다 구슬 전체 색이 단계적으로 변경된다.
등급이 높을수록 HP가 많아서 색 변경 횟수가 많고, 때리는 시간이 길어질수록 기대감이 폭발한다.

> *"아직 안 깨져?! 그러면 최소 보라 이상이다!"* — 이것이 핵심 감정.

| 등급 | 색상 | 구슬 이펙트 | 드랍 확률 (기본) |
|------|------|-----------|----------------|
| 일반 (Common) | 회색 | 없음 | 40% |
| 고급 (Uncommon) | 초록 | 약한 빛남 | 30% |
| 희귀 (Rare) | 파랑 | 빛남 + 미세 파티클 | 18% |
| 영웅 (Epic) | 보라 | 강한 빛남 + 오라 | 9% |
| 전설 (Legendary) | 금색 | 폭발적 빛남 + 번개 오라 + 화면 플래시 | 2.5% |
| 신화 (Mythic) | 주황+빛줄기 | 화면 전체 발광 + 천사 링 + 사운드 변경 | 0.5% |

### 5-2. 구슬 HP 배율 — "안 깨지면 대박"

> 핵심 원리: 등급이 높을수록 HP가 높다 → 오래 때려야 한다 → "아직도 안 깨져?!" → 기대감 폭발

**구슬 HP 배율 (일반 몬스터 HP 기준):**

| 등급 | HP 배율 | Stage 1 HP | 색 변경 횟수 | 체감 |
|------|--------|-----------|------------|------|
| 일반 | x5 | 100 | 0 (회색 → 바로 파괴) | "금방 깨지네" |
| 고급 | x8 | 160 | 1 (회색 → 초록 → 파괴) | "어 색 바뀌었다" |
| 희귀 | x12 | 240 | 2 (회색 → 초록 → 파랑 → 파괴) | "오 파랑까지!" |
| 영웅 | x18 | 360 | 3 (회색 → 초록 → 파랑 → 보라 → 파괴) | "보라다!!" |
| 전설 | x25 | 500 | 4 (회색 → ... → 금색 → 파괴) | "금색!! 금색!!" |
| 신화 | x35 | 700 | 5 (회색 → ... → 주황 → 파괴) | "미쳤다!!!" |

**구슬 HP 스케일링 공식:**

```
MonsterHP(stage) = BaseHP × (1 + (stage - 1) × 0.15)
BossHP(stage, grade) = MonsterHP(stage) × GradeMultiplier(grade)
```

**스테이지별 구슬 HP 예시 (등급별):**

| 스테이지 | 몬스터 HP | 일반(x5) | 고급(x8) | 희귀(x12) | 영웅(x18) | 전설(x25) | 신화(x35) |
|---------|----------|---------|---------|----------|----------|----------|----------|
| 1 | 20 | 100 | 160 | 240 | 360 | 500 | - (잠금) |
| 10 | 47 | 235 | 376 | 564 | 846 | 1,175 | 1,645 |

### 5-3. 색 변경 메커니즘

구슬은 항상 회색(일반)으로 시작하며, HP 데미지에 따라 등급 색상이 순차적으로 드러난다.

**색 변경 규칙:**

```
1. 스테이지 진입 시 구슬 등급 & HP 내부 결정
2. 구슬은 항상 회색(일반)으로 시작
3. HP를 (색 변경 횟수 + 1)개 균등 구간으로 분할
4. 각 구간의 HP를 깎을 때마다 다음 색으로 전환
5. **마지막 색 전환 후 최소 1.5초 유지** → 그 후 파괴 시퀀스 진입
   (파괴 직전까지 불확실성을 극대화하여 도파민 정점 보장)
```

**예시 — 영웅(보라) 등급 구슬 (Stage 1, BaseHP=20):**

```
  총 HP: 360 (20 × 18)  →  구간: 360 / 4 = 90씩

  ┌────────┬────────┬────────┬────────┐
  │ 90 HP  │ 90 HP  │ 90 HP  │ 90 HP  │
  │  회색  │  초록  │  파랑   │  보라  │
  └───┬────┴───┬────┴───┬────┴───┬────┘
      ↓        ↓        ↓        ↓
   "시작"  "어? 초록" "파랑!!" [마지막 색 1.5초 유지]
                                     ↓
                                  [파괴]
```

**색 변경 연출 상세:**

| 이벤트 | 연출 | 시간 |
|--------|------|------|
| 색 변경 순간 | 구슬 전체 0.1초 화이트 플래시 → 새 색상으로 전환 | 0.15초 |
| 색 변경 + 파티클 | 구슬 주변에 새 색상 파티클 폭발 (작은 원형) | 0.3초 |
| 색 변경 + 히트스톱 | 0.05초 미세 히트스톱 (색이 바뀌는 순간 강조) | 0.05초 |
| 색 변경 + 사운드 | 등급이 올라갈수록 점점 높은 음의 "띠링!" | 즉시 |

**핵심 설계 포인트:**
- 구슬은 항상 회색으로 시작 → 어떤 등급인지 처음엔 모름
- HP가 높아서 안 깨지면 → "이거 높은 등급이다!" 기대감 자연 발생
- 색이 바뀔 때마다 도파민 → 다음 색이 뭘지 궁금
- **일반 구슬은 금방 깨져서 실망 → 다음 스테이지 빨리 가고 싶은 욕구**
- **고등급 구슬은 때리는 시간 자체가 기대감의 원천**

### 5-4. Pity 시스템 (소프트 & 하드)

보스 구슬 드랍 전용 Pity 시스템. 가챠 Pity(섹션 4-3)와 **별도로 운영**된다.

```
// Soft Pity: 5연속 희귀 이하부터 영웅+ 확률 보정 시작
SoftPityBonus = max(0, (연속_희귀이하_횟수 - 5) × 0.05)
// 5연속 후 +5%/스택, 10연속이면 +25% 추가

// Hard Pity: 15연속 희귀 이하 시 → 다음 판정 영웅 이상 100% 보장
if (연속_희귀이하_횟수 >= 15) → 영웅 이상 확정

// 전설 전용 Pity: 40연속 전설 미등장 시 → 다음 판정 전설 이상 보장
// 신화 전용 Pity: 100연속 신화 미등장 시 → 신화 보장
```

**Pity 카운터 관리 규칙:**

| 획득 등급 | 리셋되는 카운터 | 유지되는 카운터 |
|----------|---------------|---------------|
| 영웅(Epic)+ | Soft/Hard Pity 카운터 | 전설/신화 카운터 (독립 누적) |
| 전설(Legendary) | Soft/Hard + 전설 전용 카운터 | 신화 카운터 (독립 누적) |
| 신화(Mythic) | **전체** Pity 카운터 | - |

**Pity 저장 정책:**
- 저장 타이밍: 구슬 파괴 확정(HP 0) 시점에 1회만 JSON 저장
- 색 변경 중에는 메모리에서만 관리 (I/O 미발생)
- 튜토리얼 보장 드랍 시 Pity 카운터 리셋됨
- 스테이지 재도전 시 구슬 등급 유지 (보스 방 첫 진입 시점에 결정+저장, 재도전은 동일 등급)
- 다음 스테이지 진행 시에만 새 등급 결정

**Pity 수치 요약:**

| Pity 유형 | 발동 조건 | 보장 내용 |
|----------|----------|----------|
| Soft Pity | 5연속 희귀 이하 | +5%/스택 영웅+ 확률 증가 |
| Hard Pity | 15연속 희귀 이하 | 영웅 이상 100% 보장 |
| 전설 Pity | 40연속 전설 미등장 | 전설 이상 보장 |
| 신화 Pity | 100연속 신화 미등장 | 신화 보장 |

### 5-5. 튜토리얼 보장 드랍

신규 플레이어가 구슬 등급 시스템의 재미를 자연스럽게 학습하도록 초반 스테이지에서 최소 등급을 보장한다.

| 스테이지 | 보장 등급 | 학습 목표 |
|---------|----------|----------|
| Stage 3 | 고급(초록) 이상 | "색 변경이 좋은 것" 첫 학습 |
| Stage 7 | 희귀(파랑) 이상 | "더 많이 바뀌면 더 좋다" 강화 학습 |
| Stage 15 | 영웅(보라) 1회 보장 | 첫 고등급 경험, 장기 목표 제시 |

### 5-6. 등급별 파괴 연출

| 등급 | 파괴 이펙트 | 사운드 |
|------|-----------|--------|
| 일반 | 가벼운 파편 | 퍽 |
| 고급 | 초록 파편 + 작은 빛줄기 | 파삭 |
| 희귀 | 파란 파편 + 빛줄기 3개 | 쨍 |
| 영웅 | 보라 폭발 + 화면 미세 플래시 | 쾅 + 울림 |
| 전설 | 금색 대폭발 + 화면 전체 플래시 + 번개 | 쾅쾅 + 천둥 울림 |
| 신화 | 주황 대폭발 + 슬로우모션 0.5초 + 빛기둥 + 화면 흔들림 | 전용 팡파레 |

### 5-7. 드랍 확률 스테이지 보정

스테이지가 올라갈수록 상위 등급 출현 확률이 미세하게 증가한다.

```
// 스테이지 보정 — 상위 등급에 비례 배분, 하위 등급에서 동일량 차감
TotalBonus(stage) = stage × 0.002  // Stage 100: 0.2 = 20%p 총 이동량

// 상위 3등급에 기본 확률 비율로 배분:
//   영웅 기본 9%, 전설 2.5%, 신화 0.5% → 합계 12%
//   영웅 비율 = 9/12 = 75%, 전설 비율 = 2.5/12 ≈ 20.8%, 신화 비율 = 0.5/12 ≈ 4.2%
//
//   영웅 보너스 = TotalBonus × 0.75
//   전설 보너스 = TotalBonus × 0.208
//   신화 보너스 = TotalBonus × 0.042
//
// 하위 2등급에서 동일 총량 차감 (비율 배분):
//   일반 기본 40%, 고급 30% → 합계 70%
//   일반 차감 = TotalBonus × (40/70), 고급 차감 = TotalBonus × (30/70)

// Stage 100 예시 (TotalBonus = 0.20):
//   영웅: 9% + 0.15% = 9.15%, 전설: 2.5% + 0.042% = 2.542%, 신화: 0.5% + 0.008% = 0.508%
//   일반: 40% - 0.114% = 39.886%, 고급: 30% - 0.086% = 29.914%
//   합계: 9.15 + 2.542 + 0.508 + 39.886 + 29.914 + 18(희귀) = 100.000%
//
// 주의: 희귀(Rare)는 보정 대상 아님 — 항상 18% 고정
```

**스테이지별 최대 등급 잠금 (초반 밸런스):**

```
Stage 1~9:  전설 이하만 출현 (신화 잠금)
Stage 10+:  전 등급 출현 허용
```

### 5-8. 구슬 파괴 전체 시퀀스

보스 구슬 파괴부터 장비 획득까지의 전체 타임라인.

```
[스테이지 진입] 등급 & HP 내부 결정
    ↓
[보스 방 진입] 구슬 등장 (항상 회색)
    ↓
[타격] HP 구간마다 색 변경 (회색 → 초록 → 파랑 → ...)
    ↓  변경 시마다: 플래시 + 파티클 + 사운드
    ↓  "아직 안 깨져?!" → 기대감 증폭
    ↓
[HP 0%] 최종 색상 확정 + 0.3초 히트스톱
    ↓
[파괴] 등급별 폭발 연출 + 카메라 셰이크
    ↓
[드랍] 최종 등급 색상의 장비 아이템이 구슬에서 튀어나옴
    ↓
[획득] 장비 + 골드가 캐릭터로 빨려들어옴 (0.5초)
    ↓
[전환] 획득 장비 팝업 (등급별 차등 시간) → 다음 스테이지
```

**등급별 획득 팝업 표시 시간:**

| 등급 | 팝업 시간 | 추가 연출 |
|------|----------|----------|
| 일반 / 고급 | 0.8초 | - |
| 희귀 | 1.2초 | - |
| 영웅 | 1.8초 | - |
| 전설 | 2.5초 | 팡파레 |
| 신화 | 3~4초 | 전용 풀스크린 연출 |

## 6. 장비 강화 시스템

### 6-1. 강화 단계

장비는 **+0 ~ +15**까지 강화 가능하며, 강화 단계에 따라 성공률이 하락한다.

| 강화 단계 | 성공률 | 스탯 증가율 (BaseStat 대비) | 골드 비용 | 구슬 조각 비용 | 실패 패널티 |
|----------|--------|--------------------------|----------|--------------|-----------|
| +0 → +1 | 100% | +5% | 100 | 0 | - |
| +1 → +2 | 100% | +5% | 200 | 0 | - |
| +2 → +3 | 95% | +5% | 400 | 0 | - |
| +3 → +4 | 90% | +7% | 800 | 0 | - |
| +4 → +5 | 85% | +7% | 1,500 | 0 | - |
| +5 → +6 | 80% | +7% | 3,000 | 5 | - |
| +6 → +7 | 70% | +10% | 5,000 | 10 | - |
| +7 → +8 | 60% | +10% | 8,000 | 15 | 단계 유지 (하락 없음) |
| +8 → +9 | 50% | +10% | 12,000 | 20 | 단계 유지 |
| +9 → +10 | 40% | +15% | 18,000 | 30 | 단계 유지 |
| +10 → +11 | 30% | +15% | 25,000 | 40 | **1단계 하락** |
| +11 → +12 | 25% | +15% | 35,000 | 55 | **1단계 하락** |
| +12 → +13 | 20% | +20% | 50,000 | 70 | **1단계 하락** |
| +13 → +14 | 15% | +20% | 70,000 | 90 | **2단계 하락** |
| +14 → +15 | 10% | +25% | 100,000 | 120 | **2단계 하락** |

### 6-2. 강화 스탯 계산

```
enhanceBonus = BaseStat * Sum(enhanceRatePerLevel)
```

예시: ATK 10짜리 Common 무기를 +5까지 강화 시
- 증가율 합계: 5% + 5% + 5% + 7% + 7% = 29%
- 강화 보너스 ATK: 10 * 0.29 = 2.9 (반올림 → 3)
- 최종 ATK: 10 + 3 = 13

예시: ATK 45짜리 Legendary 무기를 +15까지 강화 시
- 증가율 합계: 5%*3 + 7%*3 + 10%*3 + 15%*3 + 20%*2 + 25% = 15% + 21% + 30% + 45% + 40% + 25% = 176%
- 강화 보너스 ATK: 45 * 1.76 = 79.2 (반올림 → 79)
- 최종 ATK: 45 + 79 = 124

### 6-3. 강화 연출

| 결과 | 연출 |
|------|------|
| 성공 (+1~+9) | 장비 아이콘 금색 반짝임 + "강화 성공!" 텍스트 |
| 성공 (+10~+15) | 화면 전체 이펙트 + 카메라 셰이크 + "!!강화 대성공!!" |
| 실패 (유지) | 장비 아이콘 흔들림 + 회색 연기 이펙트 |
| 실패 (하락) | 적색 깨짐 이펙트 + 슬픈 효과음 |

### 6-4. 강화 보호권

| 아이템 | 효과 | 획득처 |
|--------|------|--------|
| 강화 보호권 (하급) | +10 이하 강화 실패 시 단계 유지 보장 | 업적, 이벤트 |
| 강화 보호권 (상급) | +11 이상 강화 실패 시 단계 유지 보장 | 보석 상점 (50보석) |

## 8. 장비 비교 및 자동 장착

### 8-1. 자동 장착 시스템

방치형 게임의 핵심 편의 기능. 플레이어가 직접 비교하지 않아도 최적의 장비를
자동으로 장착한다.

**자동 장착 로직:**

```
equipmentScore =
    (ATK * 1.0) +
    (HP * 0.3) +
    (ATK_SPEED * 50) +      // %를 소수로 변환 후 곱셈 (0.05 → 2.5)
    (CRIT_RATE * 80) +      // %를 소수로 변환 후 곱셈
    (CRIT_DMG * 40) +       // %를 소수로 변환 후 곱셈
    (MOVE_SPEED * 30) +     // %를 소수로 변환 후 곱셈
    setEffectWeight           // 세트 완성 시 보너스 점수 +100
```

- 새 장비 획득 시 현재 장착 장비와 점수 비교
- 점수가 높으면 자동 장착 (설정에서 ON/OFF 가능)
- **자동 장착 ON**이 기본값 (방치형 특성)

### 8-2. 비교 UI 사양

장비 탭에서 아이템을 탭하면 비교 팝업이 표시된다.

```
┌─────────────────────────────────────────┐
│          장비 비교                 [X]   │
│                                         │
│  [현재 장비]         [새 장비]           │
```

- 변화 없음: 회색 (-)
- 세트 효과 변동 시 경고 메시지 표시

### 8-3. 인벤토리

- 인벤토리 슬롯: 최대 **100칸** (초기 50칸, 보석으로 확장 가능)
- 정렬 옵션: 등급순, 슬롯순, 점수순, 획득순
- 필터: 슬롯별, 등급별
- 인벤토리 가득 참 시: 자동으로 가장 낮은 점수의 Common 장비 분해 (설정 가능)

## 9. 장비 분해/재활용

### 9-1. 분해 시스템

불필요한 장비를 분해하여 자원을 회수한다.

**분해 획득 자원:**

| 등급 | 골드 | 구슬 조각 | 강화석 |
|------|------|----------|--------|
| Common | 50 | 1 | 0 |
| Uncommon | 150 | 3 | 1 |
| Rare | 400 | 8 | 3 |
| Epic | 1,000 | 20 | 8 |
| Legendary | 3,000 | 50 | 20 |

**강화된 장비 분해 보너스:**
- 강화 레벨 1당 골드 +10%, 구슬 조각 +5% 추가
- 예시: +5 Rare 분해 → 골드 400 * 1.5 = 600, 구슬 조각 8 * 1.25 = 10

### 9-2. 강화석 (Enhancement Stone)

분해로 얻는 보조 자원. 강화 시 **골드 비용을 50% 감소**시키는 데 사용.

| 사용처 | 소모량 |
|--------|--------|
| +1~+5 강화 시 골드 50% 할인 | 강화석 1개 |
| +6~+10 강화 시 골드 50% 할인 | 강화석 3개 |
| +11~+15 강화 시 골드 50% 할인 | 강화석 5개 |

### 9-3. 일괄 분해

편의 기능으로 **일괄 분해**를 지원한다.

- "Common 전체 분해" 버튼
- "Uncommon 이하 전체 분해" 버튼
- 잠금(Lock) 기능: 중요한 장비에 자물쇠를 걸어 실수 분해 방지
- 장착 중인 장비는 분해 불가

## 10. ScriptableObject 스키마

### 10-1. EquipmentData (장비 정의 - ScriptableObject)

에디터에서 장비 원본 데이터를 정의하는 ScriptableObject.

```csharp
[CreateAssetMenu(fileName = "NewEquipment", menuName = "RunningRight/EquipmentData")]
public class EquipmentData : ScriptableObject
{
    [Header("기본 정보")]
    public string equipmentId;         // "WPN_001", "ARM_003" 등
    public string equipmentName;       // "동네 빨래방 방망이"
    public string description;         // 장비 설명 텍스트
    public Sprite icon;                // 장비 아이콘 스프라이트
    public EquipSlot slot;             // Weapon, Armor, Pants, Shoulder, Accessory
    public Rarity rarity;              // Common, Uncommon, Rare, Epic, Legendary, Mythic

    [Header("기본 스탯")]
    public float baseATK;
    public float baseHP;
    public float baseATK_SPEED;        // % 단위 (예: 5 = 5%)
    public float baseCRIT_RATE;        // % 단위
    public float baseCRIT_DMG;         // % 단위
    public float baseMOVE_SPEED;       // % 단위

    [Header("등급 배율")]
    public float rarityMultiplierMin;  // 해당 등급 배율 최소값
    public float rarityMultiplierMax;  // 해당 등급 배율 최대값

    [Header("특수 효과 (Legendary/Mythic)")]
    public bool hasSpecialEffect;
    public string specialEffectId;     // "dragon_tooth_blessing", "deep_sea_accel" 등
    public string specialEffectDesc;   // "공격 시 10% 확률로 추가 데미지 50% 발동"
    public float specialEffectValue1;  // 효과 수치 1 (확률, 지속시간 등)
    public float specialEffectValue2;  // 효과 수치 2 (데미지 배율, 버프량 등)

    [Header("세트 정보")]
    public string setId;               // "arad_fashion", "deep_sea", "cult_relics" (없으면 "")
}
```

### 10-2. EquipmentInstance (장비 인스턴스 - 런타임 데이터)

## 부록 A: 밸런스 노트

### 장비 파워 기여도 목표

전체 캐릭터 전투력에서 장비가 차지하는 비율 목표:

| 게임 진행 구간 | 장비 기여도 | 스탯 강화 기여도 | 스킬 기여도 |
|--------------|-----------|----------------|-----------|
| 초반 (1~50 스테이지) | 20% | 60% | 20% |
| 중반 (51~200 스테이지) | 35% | 40% | 25% |
| 후반 (201+ 스테이지) | 45% | 30% | 25% |

### 장비 확장 계획

향후 업데이트에서 추가 가능한 요소:
- **신화(Mythic) 전용 장비 데이터**: 구슬 등급 시스템에 등급 정의 완료 (섹션 5 참조), 개별 장비 아이템 추가 필요
- **각성 시스템**: 동일 장비 5개 합성으로 각성, 외형 변경 + 스탯 대폭 상승
- **코스튬 슬롯**: 전투력 무관, 외형 변경용 (과금 요소)
- **펫 장비**: 펫 시스템 도입 시 함께 추가
