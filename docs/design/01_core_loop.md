# 코어 루프 상세 기획서

> Phase 1 구현 대상 — 이 기획이 재미없으면 게임 전체가 성립하지 않는다.

## 1. 코어 루프 플로우

```
┌──────────────────────────────────────────────────────────────────┐
│                        한 스테이지 흐름                            │
│                                                                  │
│  [잡몹 방 1]  →  (잡몹 방 2)  →  [보스 방: 잡몹 + 구슬]          │
│                  ↑ 50% 확률                                      │
│                                                                  │
│  잡몹 방 내부:                                                    │
│  캐릭터 달리기 → 대규모 잡몹 무리 감지 → 전투 → 광역기로 쓸어버림  │
│  → 다음 무리 → 섬멸 → 달리기 → 반복                              │
│                                                                  │
│  보스 방:                                                         │
│  대규모 잡몹 + 개 얼굴 구슬 → 전부 파괴 → 스테이지 클리어         │
└──────────────────────────────────────────────────────────────────┘
         ↓ 스테이지 클리어
         ↓ 보상 획득 + 난이도 상승
         ↓ 다음 스테이지 자동 진입
         ↓ (무한 반복)
```

### 핵심 체험 목표
- **"학살의 쾌감"**: 10~15마리 잡몹 무리를 광역기 한 방에 쓸어버리는 핵앤슬래시 카타르시스
- **"밀도감"**: 화면에 잡몹이 바글바글, 플레이어를 향해 몰려옴 → 한꺼번에 쓰러지며 날아가는 시각적 쾌감
- **"템포"**: 방이 2~3개뿐이라 한 스테이지가 빠르게 끝남 → 성장 루프 회전 빠름
- **"성장 실감"**: 강해질수록 더 큰 무리를 한 방에 쓸어버림
- **"긴장감"**: 몬스터 접촉 데미지 → HP가 닳음 → 강화하지 않으면 사망 → 성장 동기

## 2. 캐릭터 시스템

### 2-1. 이동

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| 기본 이동속도 | 6.0 units/s | 걸어가는 느낌이 아닌, "달리는" 느낌 |
| 최대 이동속도 | 15.0 units/s | 성장 상한 (너무 빠르면 게임이 안 보임) |
| 가속 시간 | 0.15s | 정지 → 최대속도 도달 (즉각 반응감) |
| 감속 시간 | 0.1s | 몬스터 감지 시 급정거 (전투 전환 쾌적) |

**이동 규칙:**
- 항상 오른쪽으로만 이동 (플레이어 입력 없음)
- 전방에 몬스터가 없으면 달린다
- 몬스터 감지 시 감속 → 정지 → 전투 모드 전환
- 모든 몬스터 처치 후 0.3초 대기 → 다시 달리기 (여운)

**달리기 구간 연출:**
- 달리기 중 발자국 파티클 + 먼지 이펙트 (속도감)
- 몬스터 감지 거리 6.0u 진입 시: BGM이 전투 음악으로 크로스페이드
- 화면 우측 가장자리에 몬스터 실루엣이 보이기 시작 → 공격 자세 전환 애니메이션
- 이동→공격 전환: Animator `Has Exit Time = false`, `Transition Duration = 0.05`로 즉각 전환

### 2-2. 몬스터 감지

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| 감지 범위 | 4.0 units | 화면 우측 약 1/3 지점에서 감지 |
| 감지 방식 | BoxCast 전방 | 수평 방향으로 전방 탐지 |
| 감지 레이어 | "Monster" | 몬스터 레이어만 감지 |

**왜 4.0인가:** 너무 가까우면 몬스터와 겹친 후 멈춰서 어색하고, 너무 멀면 달리는 시간이 짧아 답답하다. 화면의 약 1/3 전방에서 감지하면 "몬스터가 보이자마자 전투 태세"로 자연스럽게 전환된다.

### 2-3. 기본 공격 (평타)

**콤보 구조: 3타 콤보**

| 타수 | 모션 | 데미지 배율 | 범위 | 딜레이 | 넉백 |
|------|------|-----------|------|--------|------|
| 1타 | 수평 베기 | 100% | 전방 2.0u × 1.5u | 0.25s | 0.3u |
| 2타 | 올려치기 | 110% | 전방 2.0u × 2.0u | 0.30s | 0.5u |
| 3타 | 내려찍기 | 150% | 전방 2.5u × 2.5u | 0.40s | 1.0u |

- 3타 후 0.2초 리셋 딜레이 → 1타로 복귀
- 한 콤보 사이클: 0.25 + 0.30 + 0.40 + 0.20 = **1.15초**
- DPS 계산: (100 + 110 + 150) / 1.15 = **약 313% ATK/s**

**공격 판정:**
- 각 타격은 부채꼴이 아닌 **사각형 범위** (2D 횡스크롤이므로)
- 관통: 범위 내 모든 적에게 히트 (무리 쓸어버리는 쾌감)
- 같은 타격에 같은 몬스터 중복 히트 없음

### 2-4. 데미지 공식

```
최종 데미지 = ATK × 콤보배율 × (치명타 여부 ? CRIT_DMG : 1.0)

치명타 판정: Random(0,1) < CRIT_RATE → 치명타
```

**Phase 1 기본 스탯:**

| 스탯 | 초기값 | 설명 |
|------|--------|------|
| HP | 100 | 캐릭터 체력 |
| ATK | 10 | 기본 공격력 |
| ATK_SPEED | 1.0 | 공격 속도 배율 (실제 딜레이 = 콤보 딜레이 / ATK_SPEED) |
| CRIT_RATE | 0.15 (15%) | 치명타 확률 (초반부터 치명타 연출 체감) |
| CRIT_DMG | 1.5 (150%) | 치명타 데미지 배율 |
| MOVE_SPEED | 6.0 | 이동 속도 |

> **DPS 공식:**
> - 평타 DPS = 313% × ATK_SPEED × ATK/s (치명타 미포함)
> - 돌풍베기 DPS = 350% / 5초 = 70% ATK/s
> - **실효 복합 DPS ≈ (313% + 70%) × ATK_SPEED × (1 + CRIT_RATE × 0.5) = 약 407% ATK/s** (ATK_SPEED=1.0, CRIT_RATE=15%)

### 2-5. 광역 스킬 (Phase 1 필수)

> Phase 1 핵심 체험 "학살의 쾌감"을 달성하려면, 3타 콤보(최대 2.5u 범위)만으로는 12마리 밀집 무리를 한 번에 쓸 수 없다.
> 최소 1개의 광역 스킬이 Phase 1에 반드시 포함되어야 한다.

**돌풍베기 (Gale Slash) — 자동 시전 광역기:**

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| 쿨타임 | 5.0초 | 쿨타임 기반 자동 시전 |
| 데미지 배율 | **350% ATK** | 잡몹 즉사 보장 수준 (Stage 1: 10×3.5=35 > 잡몹 HP 20) |
| 범위 | 전방 **6.0u × 3.0u** | 밀집 무리 10~15마리 동시 히트 |
| 넉백 | 2.0u | 무리 전체가 뒤로 밀려남 |
| 연출 | 대형 슬래시 이펙트 + 0.08초 히트스톱 + 카메라 셰이크 | 광역기 쾌감 강조 |

**전투 리듬:**
```
평타 콤보 (1.15초) → 평타 콤보 → 평타 콤보 → 평타 콤보 →
[쿨타임 완료] 돌풍베기 발동! (무리 10마리+ 동시 히트) →
평타 콤보 → ... 반복
```

**왜 1개 스킬인가:**
- Phase 1 검증 목적: "광역기로 쓸어버리는 쾌감"이 성립하는지 확인
- Phase 2에서 스킬 풀 확장 (2~3개 추가 스킬 + 궁극기)
- 쿨타임 자동 시전이므로 플레이어 입력 불필요 (방치형 유지)

### 2-6. 사망 & 부활

```
[HP 0] 캐릭터 사망 → 0.5초 사망 연출
    ↓
[사망 화면] 3가지 선택지 (무료 부활 소진 여부에 따라 동적 레이아웃):
    ↓
  무료 부활 남아있을 때:
    (1) [기본 CTA] 즉시 부활 (무료 N회 남음)
        → 현재 위치에서 HP 100% 회복, 3초 무적
    (2) 스테이지 재도전 (획득 골드 **전액 보존**)
    (3) 강화하기 → 강화 화면 직행 (원탭) → 강화 후 재도전 (골드 전액 보존)

  무료 부활 소진 후:
    (1) [기본 CTA] 스테이지 재도전 (획득 골드 **전액 보존**)
    (2) 강화하기 → 강화 화면 직행 (골드 전액 보존)
    (3) [유료] 즉시 부활 (다이아 5개) — 확인 다이얼로그 필수, 하단 배치

  30초 미선택 시: 자동으로 스테이지 재도전 (골드 전액 보존)

> **사망 패널티 설계 철학:** 방치형 게임에서 사망 패널티는 최소화.
> 골드/장비 손실 없음. 유일한 패널티는 "해당 스테이지 진행 시간 소실" 뿐.
> 사망은 "벌"이 아니라 "강화 유도 신호"로만 작동해야 한다.
```

**오프라인 중 사망 처리:**
- 오프라인 파밍 중 사망 시 → 자동으로 마지막 클리어 스테이지로 전환, 축소 효율로 파밍 계속
- 복귀 시 "Stage N에서 막혀서 Stage M에서 파밍했어요" 안내

## 3. 몬스터 시스템

### 3-1. 일반 몬스터

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| 기본 HP | **20** | Stage 1 기준 (광역기 즉사 보장) |
| 이동 속도 | **2.5 units/s** | 플레이어 방향으로 접근 (플레이어의 약 40%) |
| 접촉 데미지 | 2 /초 | 캐릭터와 겹쳐있는 동안 초당 피해 |
| 사망 연출 | 넉백 + 페이드아웃 | 0.5초간 날아가며 사라짐 |
| 사망 넉백 | 3타 피니셔: 3.0~4.0u | 화면 밖으로 날아가는 쾌감 |
| 드랍 | 골드 (자동 흡수, 반경 5.0u) | 사망 위치에서 플레이어로 빨려감 |

**몬스터 이동 설계:**
- 스폰 시 제자리 → 플레이어가 감지 범위에 들어오면 플레이어 방향으로 천천히 이동
- "달려드는 적 무리를 쓸어버리는" 쾌감이 정적 샌드백과는 차원이 다름
- 이동 속도 2.5u/s는 플레이어 속도(6.0u/s)의 약 40% → "달려드는 먹잇감" 느낌
- 접촉 데미지 2/초는 캐릭터 HP 100 기준 50초 생존 → 충분히 여유롭지만, 방치 시 언젠가 죽음
- 감지 후 첫 1초는 3.5u/s 돌진 → 이후 2.5u/s 감속 (위협감 + 학살감 동시 달성)

**접촉 데미지 스케일링:**
```
ContactDmg(stage) = 2 × (1 + (stage - 1) × 0.05)   // stage 1-indexed, 기존 0.1→0.05 완화
// Stage 1: 2/초, Stage 10: 2.9/초, Stage 50: 6.9/초, Stage 100: 11.9/초
// 강화하지 않으면 후반에 체력이 급격히 줄어 사망 → 강화 동기
```

### 3-2. 구슬 보스 (개 얼굴 구슬)

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| HP | **등급에 따라 다름** (아래 표 참고) | 높은 등급 = 더 두꺼운 HP |
| 이동 | 없음 (고정 오브젝트) | 방 끝에 고정 배치 |
| 크기 | 일반 몬스터의 2배 | 시각적으로 "보스" 느낌 |
| 파괴 연출 | 색 변경 → 폭발 → 장비 드랍 | 종말의 숭배자 오마주 |
| 드랍 | 골드 + **최종 색상 등급의 장비** | 구슬 색상이 곧 보상 |

### 3-2a. 구슬 등급 연출 시스템 (종말의 숭배자 오마주)

> 구슬을 타격할 때마다 구슬 전체 색이 단계적으로 변경된다.
> 등급이 높을수록 HP가 많아서 색 변경 횟수가 많고, 때리는 시간이 길어질수록 기대감이 폭발한다.
> "아직 안 깨져?! 그러면 최소 보라 이상이다!" — 이것이 핵심 감정.

**등급 & 색상 테이블:**

| 등급 | 색상 | 구슬 이펙트 | 드랍 확률 (기본) |
|------|------|-----------|----------------|
| 일반 (Common) | ⬜ 회색 | 없음 | 40% |
| 고급 (Uncommon) | 🟩 초록 | 약한 빛남 | 30% |
| 희귀 (Rare) | 🟦 파랑 | 빛남 + 미세 파티클 | 18% |
| 영웅 (Epic) | 🟪 보라 | 강한 빛남 + 오라 | 9% |
| 전설 (Legendary) | 🟨 금색 | 폭발적 빛남 + 번개 오라 + 화면 플래시 | 2.5% |
| 신화 (Mythic) | 🟧 주황+빛줄기 | 화면 전체 발광 + 천사 링 + 사운드 변경 | 0.5% |

### 3-2b. 등급별 구슬 HP — "안 깨지면 대박"

> 핵심 원리: 등급이 높을수록 HP가 높다 → 오래 때려야 한다 → "아직도 안 깨져?!" → 기대감 폭발

**구슬 HP 배율 (일반 몬스터 HP 기준):**

| 등급 | HP 배율 | Stage 1 HP | 색 변경 횟수 | 체감 |
|------|--------|-----------|------------|------|
| 일반 | ×5 | 100 | 0 (회색 → 바로 파괴) | "금방 깨지네" |
| 고급 | ×8 | 160 | 1 (회색 → 초록 → 파괴) | "어 색 바뀌었다" |
| 희귀 | ×12 | 240 | 2 (회색 → 초록 → 파랑 → 파괴) | "오 파랑까지!" |
| 영웅 | ×18 | 360 | 3 (회색 → 초록 → 파랑 → 보라 → 파괴) | "보라다!!" |
| 전설 | ×25 | 500 | 4 (회색 → ... → 금색 → 파괴) | "금색!! 금색!!" |
| 신화 | ×35 | 700 | 5 (회색 → ... → 주황 → 파괴) | "미쳤다!!!" |

**색 변경 메커니즘:**

```
1. 스테이지 진입 시 구슬 등급 & HP 내부 결정
2. 구슬은 항상 회색(일반)으로 시작
3. HP를 (색 변경 횟수 + 1)개 균등 구간으로 분할
4. 각 구간의 HP를 깎을 때마다 다음 색으로 전환
5. **마지막 색 전환 후 최소 1.5초 유지** → 그 후 파괴 시퀀스 진입
   (파괴 직전까지 불확실성을 극대화하여 도파민 정점 보장)

예시 — 영웅(보라) 등급 구슬 (Stage 1, BaseHP=20):
  총 HP: 360 (20 × 18)  →  구간: 360 / 4 = 90씩

  ┌────────┬────────┬────────┬────────┐
  │ 90 HP  │ 90 HP  │ 90 HP  │ 90 HP  │
  │  회색  │  초록  │  파랑   │  보라  │
  └───┬────┴───┬────┴───┬────┴───┬────┘
      ↓        ↓        ↓        ↓
   "시작"  "어? 초록" "파랑!!" [마지막 색 1.5초 유지]
                                     ↓
                                  [파괴]
```

**색 변경 연출 상세:**

| 이벤트 | 연출 | 시간 |
|--------|------|------|
| 색 변경 순간 | 구슬 전체 0.1초 화이트 플래시 → 새 색상으로 전환 | 0.15초 |
| 색 변경 + 파티클 | 구슬 주변에 새 색상 파티클 폭발 (작은 원형) | 0.3초 |
| 색 변경 + 히트스톱 | 0.05초 미세 히트스톱 (색이 바뀌는 순간 강조) | 0.05초 |
| 색 변경 + 사운드 | 등급이 올라갈수록 점점 높은 음의 "띠링!" | 즉시 |

**핵심 설계 포인트:**
- 구슬은 항상 회색으로 시작 → 어떤 등급인지 처음엔 모름
- HP가 높아서 안 깨지면 → "이거 높은 등급이다!" 기대감 자연 발생
- 색이 바뀔 때마다 도파민 → 다음 색이 뭘지 궁금
- **일반 구슬은 금방 깨져서 실망 → 다음 스테이지 빨리 가고 싶은 욕구**
- **고등급 구슬은 때리는 시간 자체가 기대감의 원천**

**등급별 파괴 연출:**

| 등급 | 파괴 이펙트 | 사운드 |
|------|-----------|--------|
| 일반 | 가벼운 파편 | 퍽 |
| 고급 | 초록 파편 + 작은 빛줄기 | 파삭 |
| 희귀 | 파란 파편 + 빛줄기 3개 | 쨍 |
| 영웅 | 보라 폭발 + 화면 미세 플래시 | 쾅 + 울림 |
| 전설 | 금색 대폭발 + 화면 전체 플래시 + 번개 | 쾅쾅 + 천둥 울림 |
| 신화 | 주황 대폭발 + 슬로우모션 0.5초 + 빛기둥 + 화면 흔들림 | 전용 팡파레 |

**드랍 확률 보정:**
```
// 스테이지 보정 — 상위 등급에 비례 배분, 하위 등급에서 동일량 차감
TotalBonus(stage) = stage × 0.002  // Stage 100: 0.2 = 20%p 총 이동량

// 상위 3등급에 기본 확률 비율로 배분:
//   영웅 기본 9%, 전설 2.5%, 신화 0.5% → 합계 12%
//   영웅 비율 = 9/12 = 75%, 전설 비율 = 2.5/12 ≈ 20.8%, 신화 비율 = 0.5/12 ≈ 4.2%
//
//   영웅 보너스 = TotalBonus × 0.75
//   전설 보너스 = TotalBonus × 0.208
//   신화 보너스 = TotalBonus × 0.042
//
// 하위 2등급에서 동일 총량 차감 (비율 배분):
//   일반 기본 40%, 고급 30% → 합계 70%
//   일반 차감 = TotalBonus × (40/70), 고급 차감 = TotalBonus × (30/70)

// Stage 100 예시 (TotalBonus = 0.20):
//   영웅: 9% + 0.15% = 9.15%, 전설: 2.5% + 0.042% = 2.542%, 신화: 0.5% + 0.008% = 0.508%
//   일반: 40% - 0.114% = 39.886%, 고급: 30% - 0.086% = 29.914%
//   합계: 9.15 + 2.542 + 0.508 + 39.886 + 29.914 + 18(희귀) = 100.000% ✓
//
// 주의: 희귀(Rare)는 보정 대상 아님 — 항상 18% 고정
```

**Pity 시스템 (Soft Pity + Hard Pity):**
```
// Soft Pity: 5연속 희귀 이하부터 영웅+ 확률 보정 시작
SoftPityBonus = max(0, (연속_희귀이하_횟수 - 5) × 0.05)
// 5연속 후 +5%/스택, 10연속이면 +25% 추가

// Hard Pity: 15연속 희귀 이하 시 → 다음 판정 영웅 이상 100% 보장
if (연속_희귀이하_횟수 >= 15) → 영웅 이상 확정

// 전설 전용 Pity: 40연속 전설 미등장 시 → 다음 판정 전설 이상 보장
// 신화 전용 Pity: 100연속 신화 미등장 시 → 신화 보장

// Pity 카운터 관리:
// - 영웅+ 획득 시 → Soft/Hard Pity 카운터 0으로 리셋
// - 전설 획득 시 → 전설 전용 Pity 카운터 0으로 리셋 (Soft/Hard도 리셋)
// - 신화 획득 시 → 전체 Pity 카운터 0으로 리셋
// - 각 등급 Pity 카운터는 독립 추적 (동시 누적)
// - 저장 타이밍: 구슬 파괴 확정(HP 0) 시점에 1회만 JSON 저장
// - 색 변경 중에는 메모리에서만 관리 (I/O 미발생)
// - 튜토리얼 보장 드랍 시 Pity 카운터 리셋됨
// - 스테이지 재도전 시 구슬 등급 재결정 (시드 미고정)
```

**스테이지별 최대 등급 잠금 (초반 밸런스):**
```
Stage 1~9:  전설 이하만 출현 (신화 잠금)
Stage 10+:  전 등급 출현 허용
```

**튜토리얼 보장 드랍 (상향 시퀀스):**
```
Stage 3: 무조건 고급(초록) 이상 보장 → "색 변경이 좋은 것" 첫 학습
Stage 7: 무조건 희귀(파랑) 이상 보장 → "더 많이 바뀌면 더 좋다" 강화 학습
Stage 15: 무조건 영웅(보라) 1회 보장 → 첫 고등급 경험, 장기 목표 제시
```

**구슬 파괴 전체 시퀀스 (최종):**
```
[스테이지 진입] 등급 & HP 내부 결정
    ↓
[보스 방 진입] 구슬 등장 (항상 회색)
    ↓
[타격] HP 구간마다 색 변경 (회색 → 초록 → 파랑 → ...)
    ↓  변경 시마다: 플래시 + 파티클 + 사운드
    ↓  "아직 안 깨져?!" → 기대감 증폭
    ↓
[HP 0%] 최종 색상 확정 + 0.3초 히트스톱
    ↓
[파괴] 등급별 폭발 연출 + 카메라 셰이크
    ↓
[드랍] 최종 등급 색상의 장비 아이템이 구슬에서 튀어나옴
    ↓
[획득] 장비 + 골드가 캐릭터로 빨려들어옴 (0.5초)
    ↓
[전환] 획득 장비 팝업 (등급별 차등 시간) → 다음 스테이지
        일반/고급: 0.8초, 희귀: 1.2초, 영웅: 1.8초
        전설: 2.5초 + 팡파레, 신화: 전용 풀스크린 3~4초
```

```

## 4. 방(Room) 시스템

### 4-1. 방 구조

```
←──────────────── 방 하나 (20~30 units) ────────────────→

[입장]  ···  [무리1]  ···  [무리2]  ···  [무리3]  ···  [출구]
  2u          밀집 구간      밀집 구간      밀집 구간       2u
```

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| 방 길이 | 20 ~ 30 units | 화면 약 3~5개 너비 (대량 몬스터 수용) |
| 입장 여유 구간 | 2 units | 진입 시 짧은 달리기 구간 |
| 전투 구간 | 16 ~ 26 units | 잡몹 무리 배치 영역 |
| 출구 여유 구간 | 2 units | 클리어 후 달리기 → 다음 방 전환 |

### 4-2. 몬스터 배치 — 핵앤슬래시 밀도

> 핵심: 한 무리에 몬스터가 바글바글 → 광역기 한 방에 우수수 쓰러짐

방 내부에 **밀집 무리(Horde)**를 2~3그룹 배치. 각 무리는 밀집 배치되어 광역 공격 한 번에 다수 히트.

**패턴 A: 대규모 밀집 (가장 빈번, 50%)**
```
[입장] ---- [●●●●●●●●●●●●] ---- [●●●●●●●●●●] ---- [출구]
              12마리 밀집            10마리 밀집
```

**패턴 B: 웨이브형 (30%)**
```
[입장] -- [●●●●●●] -- [●●●●●●●●] -- [●●●●●●●●●●] -- [출구]
           6마리         8마리           10마리
         (전초)        (본대)          (주력)
```

**패턴 C: 산개 + 밀집 혼합 (20%)**
```
[입장] -- [●●●] -- [●●●] -- [●●●●●●●●●●●●●●] -- [출구]
          산개       산개          14마리 밀집
        (미끼)     (미끼)       (본진 — 쓸어버리기)
```

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| 방당 잡몹 수 | 15 ~ 30마리 | 핵앤슬래시 밀도감 핵심 |
| 무리 수 | 2 ~ 3그룹 | 그룹마다 달리기 → 학살 반복 |
| 무리 내 간격 | 0.4 ~ 0.8 units | **빽빽하게** — 광역기 1회에 5~8마리 히트 |
| 무리 간 간격 | 3.0 ~ 4.0 units | 짧은 달리기로 리듬감 유지 |

**잡몹 수 스케일링:**
```
MonstersPerRoom(stage) = floor(15 + stage × 0.3)   // 최대 30 캡
```

### 왜 이 밀도인가

| 비교 | 기존 (3~8마리) | 변경 (15~30마리) |
|------|--------------|----------------|
| 타격감 | 1~2마리씩 처치 | 5~8마리 동시 처치 |
| 시각 피드백 | 데미지 텍스트 1~2개 | 데미지 텍스트 우수수 + 사체 날아감 |
| 체감 | 일반 액션 | **핵앤슬래시 학살** |
| 오브젝트 풀 | 10개면 충분 | 최소 50개 필요 (몬스터 30 + 보스방 15 + 여유) |

### 4-3. 몬스터 Y축 분포

> 2D 횡스크롤에서 밀집감은 X축(가로)뿐 아니라 Y축(세로)에서도 만들어야 "벽처럼 몰려오는" 비주얼이 완성된다.

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| Y축 분포 범위 | -1.5u ~ +1.5u (바닥 기준 3열) | 상/중/하 3줄로 분산 배치 |
| Y축 간격 | 0.8 ~ 1.2u | 서로 겹치지 않되 밀집감 유지 |

### 4-4. 방 클리어 판정

```
조건: 방 내 살아있는 몬스터 수 == 0
    ↓
클리어 연출: 0.1초 슬로우모션 → "Clear!" 배너 (0.5초, 화면 상단에서 내려옴)
    ↓
캐릭터 달리기 재개 → 출구 트리거 도달 → 다음 방 로드
```

### 4-5. 방 전환 (Room Pool 방식)

| 방식 | 설명 |
|------|------|
| 구현 | 단일 씬 내 Room Prefab 스폰/디스폰 (Additive Scene 불사용) |
| Room Pool | 최대 3개 방 유지 (현재 방, 다음 방, 대기 방) — 위치 재활용 |
| 사전 준비 | 카메라 도달 2~3초 전에 다음 방 백그라운드 생성 |
| 제거 | 카메라 뷰에서 완전히 벗어난 후 2프레임 뒤 비활성화 (GC 분산) |
| 카메라 | Cinemachine으로 캐릭터 추적, 방 전환 시 끊김 없음 |

**왜 Room Pool인가:** `Destroy` + `Instantiate`가 동일 프레임에서 발생하면 GC 히치로 프레임 드랍. Room Pool로 재활용하면 런타임 할당을 최소화.

## 5. 스테이지 시스템

### 5-1. 스테이지 구성

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| 스테이지당 방 수 | **2 ~ 3** | 빠른 스테이지 회전 |
| 잡몹 방 | 1 ~ 2개 | 대규모 잡몹 학살 |
| 보스 방 (마지막) | 1개 | 잡몹 무리 + 개 얼굴 구슬 |

**방 수 결정:**
```
RoomsPerStage(stage) = 2 + (stage % 3 == 0 ? 1 : 0)
// 기본 2방, 3의 배수 스테이지에서 3방 (약간의 변화)
```

**왜 2~3방인가:**
- 방이 많으면 스테이지 클리어까지 지루해짐
- 방치형은 스테이지 회전이 빨라야 "성장 → 보상" 루프가 만족스럽다
- 대신 방 하나에 잡몹을 대량 투입하여 밀도감으로 승부

### 5-2. 스테이지 흐름

```
스테이지 시작
  ↓
"Stage N" 표시 (등급별 차등 연출)
  ↓
[잡몹 방 1] 15~30마리 학살
  ↓
(50% 확률) [잡몹 방 2] 15~30마리 학살
  ↓
[보스 방] "WARNING!" 연출 (1.2초, 화면 dimming + 드럼 롤 사운드)
  ↓
구슬 먼저 등장 (0.3초) → 잡몹 양쪽에서 쏟아짐
  ↓
타깃 우선순위: 가장 가까운 잡몹 우선 → 잡몹 0마리 시 구슬 자동 전환
  ↓
잡몹 쓸어버리기 → 구슬 집중 타격 → 파괴
  ↓
"Stage Clear!" + 보상 (1.0초)
  ↓
다음 스테이지 즉시 진입

--- 마일스톤 연출 ---
일반 스테이지:      "Stage N" 0.8초
10의 배수 스테이지:  "Stage N" 1.5초, 골드 테두리 + 짧은 팡파레
50/100 배수:        "Stage N" 2.0초, 특별 이펙트 + "마일스톤 달성!" 서브텍스트
```
```

### 5-3. 난이도 곡선

스테이지가 올라갈수록:
- 몬스터 HP 증가 (선형: × 0.15/스테이지)
- 방당 잡몹 수 증가 (15 → 30 캡)
- 잡몹 개체가 많아 총 HP 풀이 자연스럽게 증가
- **한 방 클리어 시간이 길어짐 → 강화 동기**

**목표 클리어 타임:**

| 스테이지 | 방 수 | 방당 잡몹 | 기대 ATK | 방당 시간 | 총 시간 | 비고 |
|---------|-------|----------|---------|----------|---------|------|
| 1 | 2 | 15마리 | 10 (초기) | ~8초 | ~20초 | 온보딩 구간 |
| 5 | 2 | 16마리 | 16 | ~10초 | ~24초 | 쾌감 구간 |
| 10 | 2 | 18마리 | 25 | ~12초 | ~28초 | 첫 벽 (×1.5) |
| 20 | 2~3 | 21마리 | 45 | ~18초 | ~45초 | 성장 실감 |
| 50 | 2~3 | 30마리 | 100 | ~25초 | ~65초 | 세 번째 벽 (×2.5) |

> "기대 ATK"는 해당 스테이지까지의 골드 수입으로 ATK를 집중 강화했을 때의 추정치.
> 미강화 시 Stage 10부터 방당 30초 이상 → 사실상 진행 불가 → 강화 유도.

**설계 의도:** 초반 스테이지 20초 내 클리어 → 빠른 성장 피드백. 후반으로 갈수록 벽을 느끼며 강화/전생 유도.

**벽 구간 설계 — 계단식 HP 급등:**

> 기본 HP 스케일링은 선형이지만, 벽 구간에서 추가 배율을 적용하여 체감 난이도를 급등시킴.

```
WallMultiplier(stage):
  Stage 10: ×1.3  (첫 번째 벽 — "강화해야 하는구나" 학습, +30%)
  Stage 20: ×1.4  (두 번째 벽)
  Stage 35: ×1.5  (세 번째 벽 — 본격적 강화 유도)
  Stage 50: ×1.7  (네 번째 벽 — 전생 유도)
  Stage 80: ×1.8  (다섯 번째 벽)
  Stage 100: ×2.0 (여섯 번째 벽)
  그 외: ×1.0

실제 HP = MonsterHP(stage) × WallMultiplier(stage)
```

> **설계 원칙:** 벽 배율은 +30~70% 범위 내. 100% 초과 급등은 "학습 가능한 벽"이 아닌 "즉사 구간"이 됨. 벽 간격을 좁히고 배율을 낮춰서 빈도로 보완.

**벽 구간 사망 시 동선:**
```
벽에서 사망 → 사망 화면 (강화하기 CTA)
  ↓
강화 화면 진입 → 골드로 스탯 강화
  ↓ (골드 부족 시)
자동 파밍 모드 진입
  ↓
충분히 강화 후 벽 스테이지 재도전
```

**자동 파밍 모드 상세:**

| 항목 | 규칙 |
|------|------|
| 진입 조건 | 사망 화면에서 "파밍하기" 선택, 또는 강화 화면에서 골드 부족 시 자동 제안 |
| 파밍 스테이지 | 벽 스테이지 - 3 (안전 마진, 사망 없이 클리어 보장) |
| 파밍 중 화면 | 일반 플레이와 동일 + 상단에 "파밍 중: 목표 골드 N/M" 진행 바 |
| 종료 조건 | (1) 목표 골드 달성 시 자동 종료 + "강화 추천" 팝업, (2) 수동 중단 가능 |
| 목표 골드 | 다음 강화 1회 비용 × 3 (여유 확보) |
| 강화 접근 | 파밍 중에도 강화 화면 접근 가능 (HUD 강화 버튼) |
| 파밍 속도 표시 | "골드/분" 실시간 표시 |
| 앱 종료 시 | 오프라인 보상 시스템으로 자동 전환 (파밍 스테이지 기준 계산) |
| 몬스터/구슬 HP | 파밍 스테이지 사망 정책: 파밍 중 사망 시 스테이지 - 1로 추가 하향 |

## 6. 타격감 연출

> DNF의 핵심은 "때리는 맛". 방치형이라도 보고 있으면 기분 좋아야 한다.

### 6-1. 히트스톱 (프레임 정지)

| 상황 | 정지 시간 | 설명 |
|------|----------|------|
| 일반 타격 | 0.05초 (3프레임) | 묵직함 체감 |
| 치명타 | 0.10초 (6프레임) | 확실한 타격감 |
| 3타 (콤보 마무리) | 0.12초 (7프레임) | 콤보 피니시 강조 |
| 돌풍베기 (광역기) | 0.08초 (5프레임) | 광역 히트 강조 |
| 구슬 색 변경 | 0.05초 (3프레임) | 색 전환 순간 강조 |
| 구슬 파괴 | 0.30초 (18프레임) | 극적 연출 |

**구현 (주의사항):**
- `Time.timeScale = 0` + `WaitForSecondsRealtime` 조합 사용
- Animator는 `AnimatorUpdateMode.UnscaledTime` 설정 필수
- AudioSource는 `ignoreListenerPause = true`로 히트스톱 중에도 사운드 정상 재생
- 히트스톱 중첩 시: 남은 시간이 더 긴 쪽 우선 (큐잉 불필요, 최대값 병합)
- Cinemachine 셰이크: 히트스톱 복구 후 발동하거나, `IgnoreTimeScale = true` 설정

### 6-2. 카메라 셰이크

| 상황 | 강도 | 지속 시간 |
|------|------|----------|
| 일반 타격 | 없음 | - |
| 치명타 | 0.1 units | 0.1초 |
| 3타 마무리 | 0.15 units | 0.15초 |
| 돌풍베기 (광역기) | 0.2 units | 0.2초 |
| 구슬 파괴 | 0.4 units | 0.5초 |

### 6-3. 데미지 텍스트

| 종류 | 색상 | 크기 | 애니메이션 |
|------|------|------|-----------|
| 일반 데미지 | 흰색 | 기본 | 위로 튀어오름 + 페이드아웃 (0.6초) |
| 치명타 | 노란색 | 1.5배 | 위로 크게 튀어오름 + 흔들림 + 페이드아웃 (0.8초) |
| 보스 데미지 | 주황색 | 기본 | 위로 튀어오름 (0.6초) |

**텍스트 풀링:** 최대 60개 동시 표시, 오브젝트 풀로 관리 (대량 잡몹 동시 히트 + 치명타 대응)

### 6-4. 몬스터 피격 리액션

```
일반 타격:
  - 스프라이트 흰색 플래시 (0.05초)
  - 넉백 (타수별 다름)

사망:
  - 1~2타 사망: 수평 넉백 2.0u (0.3초)
  - 3타 피니셔 사망: 45도 위 방향 넉백 3.0~4.0u (0.4초, 화면 밖으로 날아감)
  - 광역기 사망: 방사형 넉백 (각 몬스터 다른 각도, 2.5~3.5u)
  - 날아가면서 회전 (비물리 Lerp — Rigidbody2D 사용 안 함)
  - 반투명 페이드아웃
  - 골드 드랍 파티클 생성
```

**비물리 사망 연출 — MonsterDeathBatcher:**
- 30마리 동시 사망 시 Rigidbody2D 물리 연산 부하 방지
- **프레임당 최대 8마리** Transform 보간 처리 (초과분 다음 프레임 큐잉)
- Update 루프 기반 (코루틴 아닌 단일 MonoBehaviour.Update)
- `Time.unscaledDeltaTime` 사용 (히트스톱 중에도 사망 연출 진행)
- 페이드아웃은 `MaterialPropertyBlock`으로 처리 (배칭 유지)
- 넉백 거리 80% 도달 시 조기 풀 반환 (연출 완료 대기 불필요)

### 6-5. 이펙트 파티클

| 이펙트 | 사용처 | 설명 |
|--------|--------|------|
| 슬래시 라인 | 매 타격 | 공격 방향으로 베기 자국 |
| 임팩트 버스트 | 피격 지점 | 작은 원형 폭발 |
| 치명타 스파크 | 치명타 시 | 별 모양 스파크 + 확대 |
| 구슬 크랙 | 색 변경 단계마다 강화 | 첫 색변경: 미세 크랙, 마지막 색: 심각한 크랙+진동 |
| 구슬 폭발 | 구슬 파괴 | 대형 원형 폭발 + 파편 |
| 골드 흡수 | 드랍 수집 | 금색 파티클이 캐릭터로 |

### 6-6. 사운드 설계

> 타격감의 50%는 사운드. 시각 연출만으로는 묵직함이 부족하다.

**타격음:**

| 상황 | 사운드 | 설명 |
|------|--------|------|
| 1타 (수평 베기) | 얕은 "촥" | 가벼운 슬래시음 |

| 3타 (피니셔) | 깊은 "쿵!" + 울림 | 피니셔 강조, 저음 강화 |
| 치명타 | 금속성 "쨍!" + 타격음 레이어 2중 | 일반 타격음 위에 오버레이 |
| 돌풍베기 (광역기) | "슈아앙!" + 바람 소리 + 다수 히트음 | 광역 스킬 고유음 |

- BGM: 달리기 구간 → 전투 구간 크로스페이드 (감지 거리 6.0u 진입 시)

## 7. 카메라 시스템

### 7-1. Cinemachine 2D 설정

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| Follow Offset X | +2.0 units | 캐릭터보다 살짝 앞을 비춤 (진행 방향 여유) |
| Follow Offset Y | 0 | 수평 고정 |
| Dead Zone Width | 0.1 | 약간의 놀이 허용 |
| Damping X | 0.3 | 부드러운 추적 (너무 딱딱하지 않게) |
| Damping Y | 0 | 수직 이동 없음 |
| Orthographic Size | 5.0 | 16:9 기준 화면에 10 units 높이 표시 |

**왜 Offset X +2.0인가:** 캐릭터가 화면 좌측에 위치해야 전방의 몬스터 무리가 잘 보인다. DNF도 진행 방향에 시야를 더 할당한다.

│  [캐릭터]           [몬스터들]            │

## 8. 골드 경제 시스템

### 8-1. 골드 획득

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| 일반 몬스터 드랍 | 5 + stage × 2 (1마리당) | 스테이지에 비례 |
| 구슬 보스 드랍 | (5 + stage × 2) × 10 | 일반 1마리 드랍의 10배 |
| 획득 방식 | 자동 흡수 (반경 5.0u) | 넉백 사망 시에도 놓치지 않음 |
| 방 클리어 시 | 남은 골드 전체 일괄 흡수 | 누락 방지 |
| 표시 | 화면 우상단 HUD (빠르게 올라가는 애니메이션) | 풍요감 강조 |

### 8-2. 골드 소비 — 스탯 강화 (Phase 1 최소 경제)

> 골드 사용처가 없으면 수집이 무의미. Phase 1에서도 최소 강화 시스템 필수.

**강화 메뉴 (6종):**

| 강화 항목 | 레벨당 효과 | 기본 비용 | 비용 증가율 |
|----------|-----------|----------|-----------|
| ATK 강화 | +3 ATK | 50골드 | ×1.12 /레벨 |
| HP 강화 | +15 HP | 50골드 | ×1.12 /레벨 |
| ATK_SPEED 강화 | +0.05 배율 | 80골드 | ×1.15 /레벨 | **상한 2.5** |
| CRIT_RATE 강화 | +1% | 100골드 | ×1.18 /레벨 | **상한 60%** |
| CRIT_DMG 강화 | +5% 배율 | 100골드 | ×1.18 /레벨 | **상한 300%** |
| MOVE_SPEED 강화 | +0.3 u/s | 60골드 | ×1.10 /레벨 | **상한 15.0** |

**강화 비용 공식:**
```
UpgradeCost(stat, level) = BaseCost(stat) × GrowthRate(stat) ^ level
// ATK Lv10 비용: 50 × 1.12^10 = 약 155골드
// ATK Lv30 비용: 50 × 1.12^30 = 약 1,497골드
```

**강화 밸런스 목표 (스테이지별 기대 ATK):**

| 스테이지 | 기대 ATK | 기대 HP | 방당 클리어 시간 (강화 후) | 생존 여유 |
|---------|---------|--------|------------------------|----------|
| 1 | 10 | 100 | ~8초 | ~50초 |
| 10 | 25 | 160 | ~12초 | ~35초 |
| 20 | 45 | 250 | ~18초 | ~28초 |
| 50 | 100 | 500 | ~25초 | ~17~35초 |

> 기대 ATK는 **장비 ATK 보너스 미포함** 골드 강화만 기준. 장비 착용 시 실제 ATK는 20~80% 더 높음.
> 생존 여유는 동시 접촉 몬스터 수에 따라 변동 (최악: 전원 접촉, 최선: 점진적 처치).

### 8-3. 오프라인 보상 (Phase 1 진입점)

> 방치형 게임의 Day 2 리텐션 핵심. Phase 1에서 기본 구조 구현.

```
[앱 종료 시] 마지막 접속 시각 + 클리어 가능 스테이지 저장

[앱 복귀 시]
  경과 시간 계산 (최대 8시간 캡)
    ↓
  오프라인 골드 = goldPerSecond × 경과시간(초) × 효율계수

  goldPerSecond = (방당 잡몹 수 × 잡몹 골드) / 방당 목표 클리어 시간(§5-3 테이블 참조)
  기준 스테이지 = 마지막으로 클리어한 스테이지 (벽 스테이지 아닌 실 클리어 기준)
  효율계수 = 0.1 (기본), 오프라인 중 사망 전환 시 0.05 (축소 효율)
    ↓
  "접속하지 않은 동안 XX골드 획득!" 팝업
    ↓
  [수령] → 강화 화면 유도 → 코어 루프 진입
```

### 8-4. 장비 시스템 (Phase 1 최소)

> 구슬 파괴 핵심 보상이 장비인데 효과가 없으면 구슬 시스템 전체가 공허. Phase 1에서 최소 장착 기능 필수.

**장착 슬롯: 무기 1개**

| 등급 | ATK 보너스 | 체감 |
|------|----------|------|
| 일반 | +5% ATK | 미미한 변화 |
| 고급 | +10% ATK | 약간 체감 |
| 희귀 | +20% ATK | 확실히 강해짐 |
| 영웅 | +35% ATK | 눈에 띄게 강함 |
| 전설 | +50% ATK | 스테이지가 녹음 |
| 신화 | +80% ATK | 압도적 |

**장비 처리 플로우:**
```
구슬 파괴 → 장비 드랍 팝업
  ↓
[현재 장착 vs 신규 비교] 자동 표시
  ↓
  (1) 장착 (더 높은 등급) → ATK 보너스 즉시 적용
  (2) 판매 (더 낮은 등급) → 골드로 전환 (등급별 차등)
```

**판매 가격:** 일반 50골드, 고급 100, 희귀 300, 영웅 800, 전설 2,000, 신화 5,000

> Phase 2에서 장비 강화/합성/세트 효과로 확장. Phase 1에서는 "장착 → 강해짐" 피드백만 보장.

### 8-5. 광고 보상 (Phase 1 최소 수익화)

> 수익 구조가 없으면 개발 지속 불가. Phase 1에 최소 광고 보상 2개 포함.

**광고 보상 유닛:**

| 위치 | 보상 | 빈도 | 예상 클릭률 |
|------|------|------|-----------|
| 오프라인 복귀 팝업 | "광고 보고 2배 수령" | 1회/접속 | 35~45% |
| 구슬 파괴 후 (영웅+ 등급) | "광고 보고 골드 50% 추가" | 1회/스테이지 | 60%+ (고등급) |

**하드 커런시 "다이아":**
- 광고 시청 1회 = 다이아 5개
- 즉시 부활 = 다이아 5개 (무료 3회/일 이후)
- Phase 2에서 IAP 상품 추가 (스타터 팩, 광고 제거 패스 등)

**광고 SDK 기술 정책:**
- SDK 초기화: 앱 시작 시 백그라운드 태스크로 분리 (메인스레드 블로킹 금지)
- 광고 프리로드: 스테이지 진행 중 미리 로드, 버튼은 로드 완료 후 활성화
- 로드 실패 시: "2배 수령" 버튼 dimmed + "1배 수령" 버튼만 활성화 (보상 누락 방지)
- 네트워크 없음: 광고 버튼 비노출, 1배 자동 수령으로 폴백
- 일일 광고 노출 상한: 오프라인 1회/접속, 구슬 보상 최대 10회/일

### 9-1. Pity 게이지 UI

> 숨겨진 Pity는 플레이어 동기 부여 연결고리를 잃게 한다. 간접적으로 노출.

- 보스 방에서 구슬 HP바 아래에 작은 **"행운 게이지"** 표시
- 연속 희귀 이하가 쌓일수록 게이지가 차오름
- 게이지가 가득 차면 반짝이며 "다음 보스에서 행운이 찾아옵니다!" 힌트
- Hard Pity 직전에는 게이지 색상이 금색으로 변화

## 11. 검증 기준 (Phase 1 완료 조건)

### 반드시 확인할 것
- [ ] 돌풍베기(광역기)로 10마리+ 동시 히트 시 "학살의 쾌감"이 느껴지는가
- [ ] 잡몹이 플레이어 방향으로 몰려오다 쓸려나가는 비주얼이 만족스러운가
- [ ] 잡몹이 동시에 날아가고 데미지 텍스트가 우수수 뜰 때 시각적으로 기분 좋은가
- [ ] 한 스테이지(2~3방)가 Stage 1 기준 20~30초 내에 끝나는가
- [ ] 3타 콤보 → 광역기 발동의 리듬감이 있는가
- [ ] 구슬 파괴 시 카타르시스가 있는가 (색 변경 체감, 등급별 파괴 연출)
- [ ] Stage 3에서 보장된 희귀 드랍 경험 후 등급 시스템을 이해하는가
- [ ] Stage 10~15 부근에서 "강화해야 한다"는 동기가 자연스럽게 발생하는가
- [ ] 사망 → 재시작 → 강화 → 재도전 루프가 답답하지 않은가
- [ ] 오프라인 복귀 시 보상 팝업이 "다시 켜길 잘했다" 느낌을 주는가
- [ ] 방 전환이 끊김 없이 매끄러운가
- [ ] 잡몹 30마리 동시 존재해도 프레임 드랍 없는가 (60fps 유지)

### 재미없으면 조정할 포인트
| 문제 | 조정 방향 |
|------|----------|
| 학살감이 부족하다 | 광역기 범위 확대, 잡몹 밀집도 증가 |
| 전투가 느리다 | 콤보 딜레이 축소, 광역기 쿨타임 감소 |
| 잡몹이 안 죽는다 | 잡몹 HP 하향, 관통 범위 확대 |
| 너무 순식간에 끝난다 | 무리 간 간격 증가, 방당 무리 수 증가 |
| 구슬이 재미없다 | 파괴 연출 강화, 구슬 주변 잡몹 더 배치 |
| 너무 쉽게 죽는다 | 접촉 데미지 하향, HP 강화 비용 인하 |
| 벽이 너무 높다 | 강화 비용 인하, 오프라인 보상 효율 상향 |
| 프레임 드랍 | 이펙트 LOD, 히트 배치 처리 (프레임 분산) |

## 12. Phase 2에서 추가될 요소 (참고)

아래는 Phase 1에서는 구현하지 않지만, 설계 시 확장을 고려해둘 요소:
- 추가 스킬 2~3개 + 궁극기 (스킬 풀 확장)
- 정예 몬스터 (HP 높음, 돌진/원거리 공격 등 AI 패턴)
- 장비 강화/합성/분해 시스템 (골드 + 구슬 조각 소비)
- 장비 세트 효과 (장기 수집 목표)
- 전생(Prestige) 시스템 (영구 버프)
- 치명타 외 추가 스탯 효과 (관통, 흡혈, 범위 증가 등)
