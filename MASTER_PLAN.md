# "오른쪽 달리기" 개발 마스터 플랜

> 던전앤파이터 패러디 방치형 2D 횡스크롤 액션 게임

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 오른쪽 달리기 |
| 장르 | 2D 횡스크롤 방치형 액션 (DNF 패러디) |
| 플랫폼 | Android |
| 엔진 | Unity 2022 LTS+ |
| 언어 | C# |
| 목표 해상도 | 1920x1080 (16:9 기준, Safe Area 대응) |
| 작성일 | 2026-03-30 |

## 게임 컨셉

캐릭터가 자동으로 오른쪽으로 달리면서 몬스터 무리를 호쾌하게 쓸어버리고, 마지막 방에서 개 얼굴처럼 생긴 구슬을 깨뜨린다. 이 과정을 무한 반복하며, 캐릭터를 성장시키는 방치형 게임.

## Phase 1: 코어 루프 프로토타입 (2주)

> **목표:** 캐릭터가 오른쪽으로 달리고, 몬스터를 때리고, 방을 넘기는 것까지

### 1-1. 캐릭터 기본 동작
- 자동 오른쪽 이동 (일정 속도)
- 몬스터 감지 시 정지 → 자동 공격
- 기본 공격 애니메이션 (임시 스프라이트 OK)
- Animator 상태머신: `Run → Attack → Run`

### 1-2. 몬스터 시스템
- 몬스터 프리팹 (HP, 피격 판정)
- 몬스터 무리 스폰 (방 단위로 N마리 배치)
- 피격 시 넉백 + 데미지 표시 (FloatingText)
- 사망 처리 (애니메이션 → 제거)

### 1-3. 방(Room) 시스템
- 방 = 일정 구간 (화면 2~3개 너비)
- 방 구성: `[입장] → [몬스터 무리] → [클리어 판정] → [다음 방]`
- 마지막 방: **개 얼굴 구슬** (보스 오브젝트) 등장
- 구슬 파괴 → 스테이지 클리어 → 다음 스테이지 시작 (무한 반복)

### 1-4. 스테이지 매니저

```
StageManager
├── currentStage (int)     # 현재 스테이지 번호
├── currentRoom (int)      # 현재 방 번호
├── roomsPerStage          # 스테이지당 방 수 (2~3방, 코어 루프 기획서 참조)
├── OnRoomClear()          # 방 클리어 시 다음 방 생성
└── OnStageClear()         # 구슬 파괴 시 스테이지 번호++, 난이도 상승
```

### Phase 1 완료 기준
- [ ] 캐릭터가 자동으로 뛰면서 몬스터를 처치
- [ ] 방 클리어 → 다음 방 전환
- [ ] 마지막 방에서 구슬 파괴 → 스테이지 반복
- [ ] **이것만으로 "게임"이 돌아감**

## Phase 2: 전투 시스템 고도화 (2주)

### 2-1. 스탯 시스템

```
CharacterStats (ScriptableObject)
├── ATK        # 공격력
├── ATK_SPEED  # 공격 속도
├── CRIT_RATE  # 치명타 확률
├── CRIT_DMG   # 치명타 배율
├── MOVE_SPEED # 이동 속도
└── HP         # (선택) 캐릭터 체력
```

### 2-2. 스킬 시스템
- 자동 시전 스킬 (쿨타임 기반)
- 스킬 종류 (DNF 패러디):
  - **기본 평타 콤보** (3~4타)
  - **범위 스킬** (몬스터 다수 타격, 쓸어버리는 쾌감)
  - **궁극기** (화면 전체 or 넓은 범위, 긴 쿨타임)
- 각 스킬은 ScriptableObject로 정의

### 2-3. 몬스터 다양화
- 일반 몬스터 (HP 낮음, 다수 등장)
- 정예 몬스터 (HP 높음, 약간의 공격 패턴)
- 구슬 보스 (개 얼굴 구슬, HP 높음, 파괴 연출)
- 스테이지에 따라 HP/공격력 스케일링 공식:

```
MonsterHP = BaseHP * (1 + (stage - 1) * 0.15)  // stage 1-indexed, Stage 1 = BaseHP
```

### 2-4. 타격감 연출
- 히트스톱 (상황별 0.05~0.12초, 코어 루프 기획서 참조)
- 카메라 셰이크 (스킬/궁극기 시)
- 파티클 이펙트 (타격, 치명타, 스킬)
- 데미지 폰트 (일반: 흰색, 치명타: 노란색+크게)

## Phase 3: 성장 시스템 (2주)

### 3-1. 재화 시스템

| 재화 | 획득처 | 용도 |
|------|--------|------|
| 골드 | 몬스터 처치 | 스탯 강화, 스킬 레벨업 |
| 구슬 조각 | 구슬 파괴 시 | 장비 뽑기, 특수 강화 |
| 보석 (프리미엄) | 업적/광고 | 가속, 특수 구매 |

### 3-2. 강화 시스템
- **스탯 강화:** 골드로 ATK/SPEED/CRIT 등 레벨업 (비용 점진 증가)
- **스킬 레벨업:** 골드/구슬 조각으로 스킬 데미지/쿨타임 개선
- **장비 시스템** (선택): 무기/방어구 드랍 or 뽑기

### 3-3. 전환(Prestige) 시스템
- 일정 스테이지 도달 후 **"전생"** 가능
- 전생 시: 스테이지 1로 리셋, 대신 **영구 버프** 획득
- 방치형의 핵심 리텐션 루프

## Phase 4: 방치형(Idle) 시스템 (1.5주)

### 4-1. 오프라인 보상

```csharp
// 앱 종료 시각 저장
PlayerPrefs.SetString("LastPlayTime", DateTime.UtcNow.ToString());

// 앱 재시작 시 경과 시간 계산
TimeSpan offlineDuration = DateTime.UtcNow - lastPlayTime;
float offlineGold = goldPerSecond * offlineDuration.TotalSeconds * 0.25f;
// 0.25 = 오프라인 효율 (코어 루프 기획서 참조)
```

### 4-2. 자동 전투 최적화
- 화면 밖 전투 시뮬레이션 (렌더링 스킵)
- 배터리 절약 모드 (30fps 전환, 이펙트 감소)

### 4-3. 오프라인 보상 팝업
- 앱 복귀 시 "접속하지 않은 동안 XX골드, XX구슬 획득!" UI
- 광고 시청 시 2배 보상 (선택)

## 우선순위 원칙

1. **Phase 1이 제일 중요** — 코어 루프가 재미없으면 나머지는 의미 없음
2. 아트는 **임시 에셋**(사각형, 원)으로 시작하고 나중에 교체
3. Phase 1 완성 후 직접 플레이하며 **"호쾌함"이 느껴지는지** 검증
4. 방치형 시스템은 코어 전투가 확정된 후 붙이기
